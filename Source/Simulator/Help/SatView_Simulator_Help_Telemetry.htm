Telemetry Data Generation

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ comes with a satellite simulator that can replay telemetry data or respond to predefined telecommands. This eases the development or modification of satellite databases and also supports the validation of implemented algorithms related to the post-processing of telemetry data.

How do I...

Configure the Telemetry Data Provider Device

The SatView™ Simulator can use a variety of protocols to communicate with its clients. Furthermore, it is capable to concurrently distribute telemetry data via different telemetry data provider devices simultaneously. This is achieved via the emulation of related front-end equipment by software.
Currently these devices are supported:

  • ENERTEC TT&C 3801 (Telemetry Source Packet Service only)

Before any telemetry data can be replayed the desired device must be chosen and configured correctly:

ENERTEC TT&C 3801-10 Emulation

  1. Select the Configuration... item in the Device menu or push the Configuration... button on the Device toolbar.
    The ENERTEC TT&C 3801 Emulation Dialog Box will then pop-up.
    Note:
    If the required toolbar is not visible then check the Toolbar item in the View menu.
  2. Enter the name of the device into the Name edit field.
  3. Add the IP address for the device into the IP Address edit field and adjust the TCP/IP port number in the Port spin box.
    Optional:
    • Change the buffer size and timeout interval through the Buffer and Timeout spin boxes.
  4. Specify the time interval after which the emulated device should issue a status message by marking the Issue a periodical status message every n seconds check box and enter the number of seconds n into the associated spin box.

  5. Specify the client access list

    1. Open the ENERTEC TT&C 3801 Device Clients Dialog Box by pushing onto the Clients... button.
    2. Enter the name and IP address of the client to be added to the list of those receiving full, partial or no access to the service into the Name and IP Address edit fields.
    3. Specify the access policy for the specified user by choosing one of the following options inside the Security Policy group box:

      Policy

      Description

      Allow unlimited access

      The specified client receives full access to the service.

      Restrict access from t1 to t2

      The access to the service is limited to the time interval from t1 to t2 for the client.

      Deny access

      Any attempt to access the service will be denied for the specified client.
      Note:
      Mark the Audit check box if an event message should be issued whenever a client logs-in or out.
    4. Push the Add button.
      Use the Remove button to delete a selected client from the list or Remove All to empty the access list.
    5. Mark the Allow access for non-listed users check box to allow full access as default behavior for all clients not listed.
    6. Push the OK button.
  6. Push the OK button.

Prepare the Satellite Telemetry Data

After the desired device has been selected and configured the telemetry data to be replayed has to be opened:

Loading the telemetry data

  1. Select the Open item in the File menu or push the Open button on the toolbar.
    The File Browse Dialog Box will appear.
  2. Browse to the location of the file containing the telemetry data and push the Open button.
    A new window will open containing the telemetry data to be played.

Modifying the telemetry data

The telemetry data can be modified in order to reflect different satellite conditions:

  1. Move the cursor to the location where the telemetry data should be changed and perform the necessary modifications.
  2. Select the Reload item in the Device menu or push the Reload button on the Device toolbar.
Note:
The specified changes will take place the next time the simulator replays that part of the telemetry data.

Saving the telemetry data

The telemetry data can be saved in order to keep the modifications to the telemetry data:

  • Select the Save item in the File menu or push the Save button on the toolbar.
    Alternatively, use the Save As... item in the File menu in order to save the telemetry data file to a different location.

Play the Satellite Telemetry Data

Finally, the device can be started in order to service any incoming requests:

Starting the device

  1. Activate the window those telemetry data should be distributed.
  2. Select the telemetry data provider device to be used for the data distribution by either selecting the corresponding item in the Products sub-menu of the Device menu or in the Device toolbar.
    Note:
    Make sure that the device has been configured correctly previously.
  3. Select the Play item in the Device menu or push the Play button on the Device toolbar.

Suspending the device

  1. Activate the window those telemetry data distributed should be suspended.
  2. Select the Suspend item in the Device menu or push the Suspend button on the Device toolbar.

Stopping the device

  1. Activate the window those telemetry data distributed should be stopped.
  2. Select the Stop item in the Device menu or push the Stop button on the Device toolbar.
