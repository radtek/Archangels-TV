Database Versioning Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ uses a Database Management System (DBMS) to maintain most parts of a database. More than one database can be supported per spacecraft, even local ones. Some supported database standards even provide versioning services.

How do I...

Add version information to a Database

  1. Open the Database Versioning Dialog Box if it is not open yet:
    Select the Database Versioning... item on the application button's menu.
  2. Add a version name into the Version edit field.
    Note:
    The version name as such does not need to be unique but together with the release and issue information only one entry may exist.
  3. Specify the release and issue number into the corresponding Release and Issue edit fields.
    Note:
    For some database standards this information is optional.
  4. Type an optional comment for the current version into the Comment edit field.
  5. Push the Add button to insert the current version information into the history.
    Repeat the previous steps if more than one version entry should be added.
  6. Push the OK button.
Note:

Not all database standards support versioning.

Modify version information on a Database

  1. Open the Database Versioning Dialog Box if it is not open yet:
    Select the Database Versioning... item on the application button's menu.
  2. Select the version information to be modified in the History list box.
  3. Alter the version name, release and issue number or the associated comment as requested.
  4. Push the Modify button to apply the modifications.
    Repeat the previous three steps if more than one version entry should be modified.
  5. Push the OK button.
Note:

Not all database standards support versioning.

Remove version information from a Database

  1. Open the Database Versioning Dialog Box if it is not open yet:
    Select the Database Versioning... item on the application button's menu.
  2. Select the version information to be removed in the History list box.
  3. Push the Remove button to delete the selected version information from the database.
    Repeat the previous two steps if more than one version entry should be deleted.
  4. Push the OK button.
Note:

Not all database standards support versioning.

Prerequisites:
The privilege 'Changing the database versioning history' must be enabled to change the database version history.
