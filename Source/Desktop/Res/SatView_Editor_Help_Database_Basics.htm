Database Basics

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ can support different databases standards thanks to its driver-based approach. However, the basic interface to manipulate those is always the same.

How do I...

Create a Database

  1. Open the New Dialog Box if it is not open yet:
    Select the New... item on the application button's menu.
  2. Select the Database item in the Type combo box in order to request a new database to be created.
  3. Type the name of the database into the Name edit field inside the Database Properties group box.
  4. Select or enter the name of the spacecraft this database is dedicated to in the Spacecraft combo box.
  5. Choose the server on which the database should be created by selecting its name in the Server combo box.
  6. Specify the scope of the database by choosing between the Global (only one per spacecraft) or Local (multiple instances allowed) radio button in the Characteristics group box.
  7. Choose one of the supported database standards by selecting the appropriate items in the Standard as well as in the Implementation combo boxes.
  8. Type the name of the binary file becoming the compiled database image into the Filename edit field inside the Files group box. Use the ... button in order to find an appropriate directory.
  9. Push the Details... button to specify some more details about the database. The DBMS Settings Dialog Box will appear:

    Configuring & Optimizing the DBMS Settings

    1. Check the Data radio button in order to indicate that all following information is related to the data file of the database.
    2. Enter the name of the file into the Filename edit field.
    3. Specify the directory where the file should be located by typing it into the Location edit field. Use the Browse... button in order to find a suitable directory.
    4. Determine the initial, maximal and grow size of the file by entering the various sizes (in MB) into the associated Initial Size, Maximal Size and Grow Size edit fields.
      Note:
      Mark the Unlimited check box if a maximal size for the file is not applicable. Use the Unchangeable check box if the selected database file should be limited in its growth.
      The Read-only check box can be marked if the database should be 'read-only'.
    5. Check the Logs radio button and repeat the last steps (except for the first one) in order to specify the same information for the logs file of the database.
    6. Push the OK button.

    Browsing the Database Tables

    1. Push the Tables... button to view the relational tables to be created for the database. The Database Tables Dialog Box will appear:
    2. Browse through the tables by selecting them in the Table combo box. This displays their columns and format in the Content listbox.
    3. Push the OK button.
    Note:

    This option is only available for relational databases.
  10. Push the Users... button to specify the users that are allowed to access and modify the database. The User Accounts Dialog Box will appear:

    Creating a new User Account

    Each SatView™ database holds a list of privileges for each user describing his permissions on that database. The set of privileges associated with a user is contained in a user account.

    1. Enter the name of the user account to be created into the Users edit field.
    2. Push the Add... button. The User Privileges Dialog Box will appear.
    3. Enter the password into the Use edit field of the Password group box.
    4. Confirm the password by filling it in again in the Confirm edit field.
    5. Select the privileges to be granted to the current user in the Rights combo box inside the Permissions group box.
    6. Push the Allow button.
      Repeat the previous two steps as many times as privileges should be granted.
    7. Push all necessary OK buttons.

    Modifying a User Account

    1. Select the user those account password or permissions on the database is going to be changed in the Users combo box.
    2. Push the Change... button. The User Privileges Dialog Box will appear and then change either the user's password and/or privileges.
    3. Push all necessary OK buttons.

    Deleting a User Account

    1. Select the user those account has to be deleted in the Users combo box.
    2. Push the Remove button.
    3. Push the OK button.
  11. Push the Validity... button to specify how long the database should be considered to be valid. The Database Validity Dialog Box will appear:

    Specifying the Database Validity

    SatView™ databases can have an attached validity period i.e. trying to operate on an expired database will be alerted and real-time telemetry data processing (including archiving) will consequently be stopped. Retrieving historical telemetry data, however, or submitting telecommands is still be possible in those cases.

    1. Check either the Unlimited or Between t1 and t2 radio button depending on the requirement to assign an unlimited validity to the database or not. For the latter case specify t1 and t2 by typing the dates into the corresponding edit fields.
    2. Push the OK button.

    Using Database Versioning

    Some database standards supported by SatView™ use versioning. This allows for a differentiation between multiple database versions and for consistency checks when importing data (e.g. saved telecommand stacks).

    Adding version information to a Database

    1. Open the Database Versioning Dialog Box if it is not open yet:
      Select the Database Versioning... item on the application button's menu.
    2. Add a version name into the Version edit field.
      Note:
      The version name as such does not need to be unique but together with the release and issue information only one entry may exist.
    3. Specify the release and issue number into the corresponding Release and Issue edit fields.
      Note:
      For some database standards this information is optional.
    4. Type an optional comment for the current version into the Comment edit field.
    5. Push the Add button to insert the current version information into the history.
      Repeat the previous steps if more than one version entry should be added.
    6. Push the OK button.
    Note:

    Not all database standards support versioning.

    Modifying version information on a Database

    1. Open the Database Versioning Dialog Box if it is not open yet:
      Select the Database Versioning... item on the application button's menu.
    2. Select the version information to be modified in the History list box.
    3. Alter the version name, release and issue number or the associated comment as requested.
    4. Push the Modify button to apply the modifications.
      Repeat the previous three steps if more than one version entry should be modified.
    5. Push the OK button.
    Note:

    Not all database standards support versioning.

    Removing version information from a Database

    1. Open the Database Versioning Dialog Box if it is not open yet:
      Select the Database Versioning... item on the application button's menu.
    2. Select the version information to be removed in the History list box.
    3. Push the Remove button to delete the selected version information from the database.
      Repeat the previous two steps if more than one version entry should be deleted.
    4. Push the OK button.
    Note:

    Not all database standards support versioning.

    Prerequisites:
    The privilege 'Changing the database versioning history' must be enabled to change the database version history.
  12. Push the OK button.

Prerequisites:
The privileges 'Creating databases' or 'Creating (remote) databases' must be enabled.

Open a Database

  1. Open the Database Open Dialog Box if it is not open yet:
    Select the Open Database... item on the application button's menu.
  2. Choose the server on which the database to be opened is located by selecting it in the Server combo box.
  3. Determine the spacecraft to which the database belongs to in the Spacecraft combo box.
  4. Select one of the databases available for the desired spacecraft on the specified server by highlighting it in the list box inside the Databases group box.
    Note:
    Details about the selected database are displayed in the Characteristics group box.
  5. Push the OK button.

Update a Database

Databases within SatView™ contain many different components, like tables, displays or procedures, most of them interlinked with each other. Depending on the nature of those components, various customizable user interfaces are available that support the database maintenance in a comprehensive way. In order to guarantee the correctness and consistency of a database it has to undergo a parsing & compilation pass before it finally can be used in the operational environment.

  • Working with Database Components

    The database components can be divided into two types: Those stored in the relational Database Management System (DBMS) and others organized as executable modules.

    Editing Database Component Items

    The various database component types have to be edited in different ways: Whereas all core DBMS components share a form-based interface, the executable components require individual editors.

    1. Open the database component to which the item to be edited belongs to by selecting the Open... è Database Component... item on the application button's menu. A dialog box will appear allowing the selection of the requested database item. The same result can be achieved by expanding the corresponding component tree in the Database page of the Workspace pane and double-clicking onto the desired entry.
    2. Editing differs depending on the component type:

      Core DBMS Components

      The core DBMS components can be browsed and edited within forms showing all database tables relevant to the selected component. Within each form there is always a top-level table owning zero or more parental or non-parental sub-tables connected in a way showing their relationship. Some of the (top-level) tables can contain sub-tables that require the relevant data to be filled into their sub-tables to make them complete. There is also a difference in manipulating database items in a top-level table compared to the same actions for a parental sub-table: Actions like saving, printing or deleting that need to be applied to a top-level table are performed through the application button's menu whereas the Database è Tables ribbon bar category (or the sub-table floating menu) has to be used to edit sub-tables.

      Executable Components

      Some components of the database must exist as executable files and hence are not part of the DBMS. They require a different interface for editing.

      • Mimics Displays

        A Mimics Display (MMD) is a set of dynamic graphic objects that are animated by telemetry data. A custom language, the Mimics Description Language (MDL) has been developed to control the mimics objects. Individual areas of the display can be linked with other MMDs allowing a kind of zoom-in functionality.
        The editing interface is built of two panes where the upper one contains the graphical area and the lower one the coding area.

        • Understand the Layout
        Note:

        Consult the chapter about the design of Mimics Displays for more information.

        Automation Scripts

        An Automation Script is a piece of code executing third-party software (in a invisible way) and taking advantage of its capabilities. Currently, the JScript .NET® language is available for the implementation of such code. There is also a programmatic access to telemetry data.
        The editing interface is built of two panes, the upper one containing the coding area and the lower one showing the results (i.e. output) of the script.

        • Understand the Layout
        Note:

        Consult the chapter about the design of Automation Scripts for more information.

        Telecommand Procedures

        A Telecommand Procedure is executable code that can issue telecommands depending on various conditions including those found in housekeeping telemetry data. There is a special language called the Telecommand Procedures Language (TPL) to implement such modules.
        The editing interface is built of two panes, the left one providing space to graphically design the programmatic flow of the procedure and the right one consisting of a property sheet to enter details about the various elements constituting the procedure.

        • Understand the Layout
        Note:

        Consult the chapter about the design of Telecommand Procedures for more information.

    Deleting Database Component Items

    Database components are either part of the core DBMS or exist as executable modules. This difference is also reflected in the way how database items have to be deleted.

    1. Open the database component to which the item to be deleted belongs to by selecting the Open... è Database Component... item on the application button's menu. A dialog box will appear allowing the selection of the requested database item. The same result can be achieved by expanding the corresponding component tree in the Database page of the Workspace pane and double-clicking onto the desired entry.
    2. Deleting may differ depending on the component type:

      Core DBMS Components

      • Select the Delete item in the application button's menu to remove the database item.

      Executable Components

      • Select the Delete item in the application button's menu to remove the database item.

    Prerequisites:
    The corresponding privilege must be enabled to create, modify or delete a database component item.
    See Privileges Summary for all existing privileges.

    Compiling the complete Database or Components of it

    It is recommended to compile the complete database after any modifications have taken place. However, due to time saving considerations, it is also possible to choose individual components of a database to perform a syntax check or compilation. In addition, batch profiles can be defined to store different types of parsing or compilation passes.

    • Configuring the Compile Options

      While compiling database component items third-party software is used to generate executable modules. By configuring the compile options it is possible to customize that process.

      Specifying the Compiler & Linker

      1. Open the Compile Options Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Options button inside the Settings panel.
      2. Enter the path of the executable file to be used as compiler into the Compiler edit field.
        Use the Browse... button in order to locate the file if necessary.
      3. Type the path of the linker program into the Linker edit field.
      4. Specify the directory where the necessary include files can be found by entering that location into the Include Directory edit field.
        Eventually use the Browse... button in order to locate the directory.
      5. Do the same for the location where all library files reside by typing it into the Library Directory edit field.
        Again use the associated Browse... button if necessary.
      6. Push the OK button.

      Customizing the Optimizations & Warning Behavior

      1. Open the Compile Options Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Options button inside the Settings panel.
      2. Inside the Directives group box choose the component for which the compile optimizations and report behavior should be changed in the Use the compiler options below for the Component combo box.
      3. Choose the desired warning level in the Warning Level combo box.
        The following choices are available:

        Severity

        Description

        No Warnings

        Does not list any warnings at all.

        Only severe

        Shows only the most severe warnings.

        Less severe

        Shows all warnings starting from a quite low severity.

        Almost all

        Shows all warnings but not informational messages.
        Mark the Treat all warnings as errors check box if a compilation should be considered as failed even when only warnings were issued.
      4. Specify how the generated code should be optimized by choosing one of the following options:

        Option

        Description

        None

        Disables all optimizations.

        Default

        Performs a default optimization on the compiled code. It depends on the compiler used what type of performance options are implemented for this option.

        Maximize Speed

        Optimizes for maximum speed.
        This is the recommended option.

        Minimize Size

        Minimizes the size of the compiled code.
      5. Push the OK button.

      Prerequisites:
      The privilege 'Changing the database compile options' must be enabled to configure the compile options.

      Specifying Compile Directives

      Compile directives profiles are a useful tool to suppress certain messages and to further customize the error & warning behavior during compilation.

      Creating a new Compile Directives Profile

      1. Compile the database components first before creating or changing the compile directives.
        The resulting messages will appear in the Build tab of the Output tool window.
      2. Open the Compile Directives Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Directives button inside the Settings panel.
      3. Enter the name of the directives profile to be created into the Profile edit field.
      4. Select those messages in the Messages list box that need to be suppressed in future compilations when this profile is applied and then push the Disable button.
        Note:
        When messages need to be re-enabled they must be selected in the Disabled list box first before using the Enable button.
        Use the Delete button to remove a disabled message which does not need to be re-enabled any more because the cause for its occurrence has been resolved.
        Mark the Do not show any warnings check box if no warnings should be listed when a database is compiled using the selected compile directives profile.
      5. Push the Add button.
        Repeat the last three steps in order to add more than one compile directives profile.
      6. Push the OK button.

      Deleting a Compile Directives Profile

      1. Open the Compile Directives Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Directives button inside the Settings panel.
      2. Select the name of the directives profile to be deleted inside the Profile combo box.
      3. Push the Remove button.
        Use the Remove All button in order to delete all compile directives profiles.
        Note:
        The Default compile directives profile cannot be deleted.
      4. Push the OK button.

      Applying a Directives Profile to a Compile Pass

      1. Open the Compile Directives Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Directives button inside the Settings panel.
      2. Select the name of the directives profile to be applied to the next compilation pass inside the Profile combo box.
      3. Push the OK button.

      Using Compile Batch Profiles

      Parse or compile batch profiles are a convenient tool to manage different database verification and translation processes. Their execution can be scheduled on demand.

      Creating a new Batch Profile

      1. Open the Compile Batches Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Batches button inside the Compilation panel.
      2. Enter the (unique) name of the batch profile to be created into the Profile edit field.
      3. Inside the Properties group box select the database components to be parsed or compiled with this batch profile by marking the appropriate check boxes.
      4. Specify whether a full compilation or only a syntax check should take place on the selected database components by checking either the Compilation or Syntax Check radio button inside the Action group box.
      5. Decide whether the batch profile should be executed immediately, at a certain time or daily by checking either the Immediately, At Time t or Daily at t radio button inside the Schedule group box.
      6. Push the Add button.
        Repeat the previous steps (except for the first one) if more than one batch profile should be created.
      7. Push the OK button.

      Modifying a Batch Profile

      1. Open the Compile Batches Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the batches button inside the Compilation panel.
      2. Select the name of the batch profile to be modified inside the Profile combo box and perform the necessary changes.
        Repeat this step as many times as profiles need to be modified.
      3. Push the OK button.

      Deleting a Batch Profile

      1. Open the Compile Batches Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Batches button inside the Compilation panel.
      2. Select the name of the batch profile to be deleted inside the Profile combo box.
      3. Push the Remove button. Use the Remove All button in order to delete all compile batches.
        Note:
        The Default batch profile cannot be deleted.
      4. Push the OK button.

      Prerequisites:
      The privilege 'Changing the database compile batches' must be enabled to create, modify or delete compile batch profiles.

    Directly parsing or compiling the:

    • Complete Database

      • Push the Compile All button inside the Compilation panel of the Build ribbon bar category.
      Note:

      All the results related to the parsing or compiling process are listed inside the Build tab of the Output tool window.

      Database Components

      1. Open the Compile Batches Dialog Box if it is not open yet:
        Select the Build ribbon bar category and push the Batches button inside the Compilation panel.
      2. Choose the Default profile in the Profile combo box. This prevents an interference with other batch profiles.
      3. Inside the Properties group box select the database components to be parsed or compiled by marking the appropriate check boxes.
      4. Specify whether a full compilation or only a syntax check should take place on the selected database components by checking either the Compilation or Syntax Check radio button inside the Action group box.
      5. Check the Immediately radio button within the Schedule group box to start the execution of the default batch as soon as possible.
      6. Push the OK button.
      Note:

      All the results related to the parsing or compiling process are listed inside the Build tab of the Output tool window.

    The privilege 'Compiling the database' must be enabled to parse or compile a database or database component.

Search a Database

Efficient database changes preclude a tool with which a database can be searched for parameters, displays and other component items. The most common queries are contained in the corresponding pages. Complex searches can be executed by using the DBMS language directly.

  • Searching core DBMS Database Components

    1. Open the Database Find Dialog Box if it is not open yet:

    Searching other Database Components

    Database components that are not part of the core DBMS provide different search interfaces. Most of them have associated source code that can be searched, sometimes even the capability to search for graphical elements.
    Refer to the individual chapters about those database components to learn more how to search them.

Delete a Database

  1. Open the Database Delete Dialog Box if it is not open yet:
    Select the Delete Database... item on the application button's menu.
  2. Choose the server on which the database to be deleted is located by selecting it in the Server combo box.
  3. Determine the spacecraft to which the database belongs to in the Spacecraft combo box.
  4. Select one of the databases available for the desired spacecraft on the specified server by highlighting it in the Databases combo box.
    Note:
    Details about the selected database are displayed in the Characteristics group box.
  5. Push the OK button.

Prerequisites:
The privilege 'Deleting (remote) databases' must be enabled to delete databases.

Import Databases

A local copy of a database can easily be created by importing it from a valid source. However, it must be stated that these replications are not automatically synchronized.

  1. Open the Database Import Dialog Box if it is not open yet:
    Select the Import Database... item on the application button's menu.
  2. Choose the server on which the database to be imported is located by selecting it in the Server combo box.
  3. Determine the spacecraft to which the database belongs to in the Spacecraft combo box.
  4. Select one of the databases available for the desired spacecraft on the specified server by highlighting it in the Database combo box.

    Importing a complete database

    This procedure imports a complete database with a different name.

    1. Enter the name of the imported database into the Import to edit field inside the Destination group box.
    2. Specify the directory where the files of the database should be moved to by typing its path into the Location edit field.
      Note:
      Use the Browse... button to search for a directory.
    3. Push the Details... button to specify some more details about the database.
      The DBMS Settings Dialog Box will appear:

      Configuring & Optimizing the DBMS Settings

      1. Check the Data radio button in order to indicate that all following information is related to the data file of the database.
      2. Enter the name of the file into the Filename edit field.
      3. Specify the directory where the file should be located by typing it into the Location edit field. Use the Browse... button in order to find a suitable directory.
      4. Determine the initial, maximal and grow size of the file by entering the various sizes (in MB) into the associated Initial Size, Maximal Size and Grow Size edit fields.
        Note:
        Mark the Unlimited check box if a maximal size for the file is not applicable. Use the Unchangeable check box if the selected database file should be limited in its growth.
        The Read-only check box can be marked if the database should be 'read-only'.
      5. Check the Logs radio button and repeat the last steps (except for the first one) in order to specify the same information for the logs file of the database.
      6. Push the OK button.

      Browsing the Database Tables

      1. Push the Tables... button to view the relational tables to be created for the database. The Database Tables Dialog Box will appear:
      2. Browse through the tables by selecting them in the Table combo box. This displays their columns and format in the Content listbox.
      3. Push the OK button.
      Note:

      This option is only available for relational databases.
    4. Push the Users... button to specify the users that are allowed to access and modify the database.
      The User Accounts Dialog Box will appear:

      Specifying the Database Users

      Each SatView™ database holds a list of privileges for each user describing his permissions on that database. The set of privileges associated with a user is contained in a user account.

      1. Enter the name of the user account to be created into the Users edit field.
      2. Push the Add... button. The User Privileges Dialog Box will appear.
      3. Enter the password into the Use edit field of the Password group box.
      4. Confirm the password by filling it in again in the Confirm edit field.
      5. Select the privileges to be granted to the current user in the Rights combo box inside the Permissions group box.
      6. Push the Allow button.
        Repeat the previous two steps as many times as privileges should be granted.
      7. Push all necessary OK buttons.
    5. Push the Validity... button to specify how long the database should be considered to be valid.
      The Database Validity Dialog Box will appear:

      Specifying the Database Validity

      SatView™ databases can have an attached validity period i.e. trying to operate on an expired database will be alerted and real-time telemetry data processing (including archiving) will consequently be stopped. Retrieving historical telemetry data, however, or submitting telecommands is still be possible in those cases.

      1. Check either the Unlimited or Between t1 and t2 radio button depending on the requirement to assign an unlimited validity to the database or not. For the latter case specify t1 and t2 by typing the dates into the corresponding edit fields.
      2. Push the OK button.

    Importing the contents of another database

    This procedure imports the contents of a database into an existing one.

    1. Mark the Data only check box.
    2. Select the database to import the data inside the Import to combo box.
    Note:

    The users and the validity of an existing database cannot be modified when importing the contents of a database. Use the commands inside the Security panel of the Home ribbon bar category instead.

    Importing database files

    This procedure imports the data of database files.

    1. Mark the Data only check box as well as the Data Files check box.
    2. Enter the directory of the database files to be imported by typing the path into the Location edit field.
      Note:
      Use the Browse... button to search for the data files directory.
    3. Add a prefix string into the Prefix edit field and the file extension into the Extension edit field to form the complete file name of the data files.
      Note:
      The data files must have the following name: 'Location\PrefixTABLE.Extension' to import successfully.
      If 'Location\Prefix.Extension' forms an existing database data file, only that one is imported.
    4. Specify the string delimiter used by the data files inside the String Delimiter combo box. Do the same for the field and row delimiters by choosing the appropriate character inside the Field and Row combo box.
    5. Select the database to import the data inside the Import to combo box.
    Note:

    The users and the validity of an existing database cannot be modified when importing database files. Use the commands inside the Security panel of the Home ribbon bar category instead.
  5. Push the OK button.

Related Topics:

Import Database Component Items

The capability to synchronize distributed databases is an important feature and also results in an interface to import or export individual database items.

  1. Open the Database Component Import Dialog Box if it is not open yet:
    Select the Import è Database Component... item on the application button's menu.
  2. Enter the name of the file that contains the database component items to be imported into the Filename edit field.
    Use the Browse... button in order to search for the file.
  3. Select the items to be imported inside the list box and specify how to import them by either checking the Default or Custom radio button. By using the Custom radio button it is possible to import a single database item with another name and description then when it was exported.
    Note:
    Importing a database item with the same name than an existing one will overwrite it.
  4. Push the OK button.

Export Database Component Items

The capability to synchronize distributed databases is an important feature and also results in an interface to import or export individual database items.

  1. Open the Database Component Export Dialog Box if it is not open yet:
    Select the Export è Database Component... item on the application button's menu.
  2. Choose the database component items to be exported in the Directory list box.
  3. Enter the name of the file to contain the database items to be exported into the Filename edit field.
    Use the Browse... button in order to find an existing file.
  4. Specify how to export the items by either checking the Default or Custom radio button. By using the Custom radio button it is possible to export a database single item with a different name and description.
  5. Push the OK button.

Prerequisites:
The privileges 'Creating databases' or 'Creating (remote) databases' must be enabled to import databases.

Export Databases

  1. Open the Database Export Dialog Box if it is not open yet:
    Select the Export Database... item on the application button's menu.
  2. Choose the server from which the database to be exported is located by selecting it in the Server combo box.
  3. Determine the spacecraft to which the database belongs to in the Spacecraft combo box.
  4. Select one of the databases available for the desired spacecraft on the specified server by highlighting it in the Database combo box.
  5. Mark the Export all tables check box if all tables of the selected database should be exported to files or check Only if a subset of tables should be exported. In the latter case, select a table in the right list box and push the < button in order to add it to the list of tables to be exported. Repeat the last step as many times as tables need to be exported.
    Use the > button to remove a selected table inside the left list box (foreseen to be exported) or push the >> to delete all of them.
  6. Enter the destination directory of the export files into the Location edit field.
    Note:
    Use the Browse... button to search for an existing directory.
  7. Type a file name prefix into the Prefix edit field if the resulting file names should be built with this prefix. In addition, it is recommended to enter a file extension into the Extension edit field.
    The file name is built as follows: Location\PrefixTable_Name.Extension
    Note:
    The prefix and extension can be left empty.
  8. Inside the Format group box choose the string, field and row delimiters to be used when the database table is exported. Just select the appropriate items in the String Delimiter, Field and Row combo boxes.
  9. Push the Unicode check box if the exported file should be encoded in Unicode (and not as an ASCII file).
  10. Push the OK button.
