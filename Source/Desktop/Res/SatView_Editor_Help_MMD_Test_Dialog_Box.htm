MMD Test Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ provides a convenient way how mimics displays can be tested before they are used in an operational environment. This is achieved by predefining telemetry conditions to be tested and then executing the associated source code.

How do I...

Test the Mimics Display

The simplest testing scenario consists of initializing a telemetry parameter (driving a mimics object) with a certain value and then executing the code. The result caused by the code is then rendered directly on the image. More complex setups are possible too by using test profiles or scripts.

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select one of the telemetry parameters used to control the behavior of one or more mimics objects in the Parameter combo box.
  3. Choose the appropriate calibration table for the telemetry parameter (if any) in the Table combo box.
    Note:
    Some of the telemetry parameters may have more than one calibration table associated with it. Select the correct one in order to guarantee an expected mapping from raw to calibrated values.
  4. Enter or select a raw value for the current parameter in the Value combo box.
    Note:
    Telemetry parameters with an associated textual calibration table list all available values in the Value combo box.
  5. Select the required occurrence of the telemetry parameter to be assigned with the previously specified value.
    Note:
    Past values cannot be specified.
  6. Specify the out-of-limit condition in which the telemetry parameter should be by checking either the No OOL, Hard OOL, Soft OOL, Delta OOL or Consistency OOL radio button.
  7. Repeat the same with the data quality by marking the Invalid Data and/or Bad Data check box.
    Note:
    Indicating invalid data implies a false validity. Bad data on the other hand indicates that its value might be corrupted.
  8. Push the Set Value button in order to initialize the telemetry parameter with the specified value, data quality and out-of-limit condition.
    Repeat the previous steps (except for the first one) if more than one telemetry parameter should be associated with a value.
  9. Push the Test button as soon as all involved telemetry parameters are initialized with a value and watch the changes on the image of the mimics display.
    Note:
    Use the Stop button in order to discontinue any blinking or other automatic (periodic) behavior or the Reset button to revert the display to its original state.
  10. Push the OK button.

Prerequisites:
The privilege 'Creating and modifying mimics displays' must be enabled to test mimics displays.

Create a new Test Profile

The test profiles contain one or more telemetry parameters initialized with a certain value. They are used to simulate certain telemetry conditions and can also be part of a test script.

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Enter the name of the profile to be created into the Name edit field of the Profiles group box.
  3. Initialize a telemetry parameter with a value, data quality and out-of-limit condition by making the appropriate selections and push the Set Value button.
  4. Push the << button in order to add the current telemetry parameter to the profile. Repeat the last two steps as many times as parameters should be inserted into the profile.
    Use the >> button in order to remove one from the profile.
  5. Push the Add button. This stores the test profile.
  6. Push the OK button.

Prerequisites:
The privilege 'Creating and modifying mimics displays' must be enabled to test mimics displays.

Use Test Profiles

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select the profile to be applied in the Name combo box.
  3. Push the Test button. This will first initialize all telemetry parameters that are part of the profile with those values, then execute the code and render the image accordingly.
    Repeat the previous two steps if more than one profile should be applied.
  4. Push the OK button.

Prerequisites:
The privilege 'Creating and modifying mimics displays' must be enabled to test mimics displays.

Delete a Test Profile

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select the profile to be deleted in the Name combo box.
  3. Push the Remove button.
  4. Push the OK button.

Prerequisites:
The privilege 'Creating and modifying mimics displays' must be enabled to test mimics displays.

Use Test Scripts

Test scripts are way to replay predefined telemetry conditions. They can consist of initialized telemetry parameters or test profiles separated by a definable time interval.

  1. Open the MMD Test Dialog Box if it is not open yet:
    Push the Test button inside the Debugging panel of the Build ribbon bar category.
  2. Select the telemetry parameter to be inserted into the script in the Parameter combo box. Initialize it and push the Set Value button. Use the << button to finally insert it into the script.
    If a test profile should be inserted instead of a telemetry parameter, choose its name in the Name combo box inside the Profiles group box and push the < button.
    Repeat the previous step as many times as telemetry parameters or test profiles should be added to the script.
  3. Adjust the delay to be inserted between the various steps by filling the correct number of milliseconds into the Delay edit field. Use the > button to remove a selected entry from the script.
  4. Push the Run button to start executing the script.
    Note:
    Use the Stop button in order to discontinue the execution of the script. The Reset button is available to revert the image to its original state which might be useful when it is saved afterwards.
  5. Push the OK button.

Prerequisites:
The privilege 'Creating and modifying mimics displays' must be enabled to test mimics displays.
