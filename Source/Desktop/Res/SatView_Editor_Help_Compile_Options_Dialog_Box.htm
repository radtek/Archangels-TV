Compile Options Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ uses third-party software to generate executable modules from certain database component items. By configuring the compile options it is possible to customize that process.

How do I...

Specify the Compiler & Linker

  1. Open the Compile Options Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Options button inside the Settings panel.
  2. Enter the path of the executable file to be used as compiler into the Compiler edit field.
    Use the Browse... button in order to locate the file if necessary.
  3. Type the path of the linker program into the Linker edit field.
    Note:
    Mark the 32-bit check box to indicate that the specified compiler and linker are targeting a 32-bit platform (the default is 64-bit).
  4. Specify the directory where the necessary include files can be found by entering that location into the Include Directory edit field.
    Eventually use the Browse... button in order to locate the directory.
  5. Do the same for the location where all library files reside by typing it into the Library Directory edit field.
    Again use the associated Browse... button if necessary.
  6. Push the OK button.

Prerequisites:
The privilege 'Changing the database compile options' must be enabled to configure the compile options.

Customize the Optimizations & Warning Behavior

  1. Open the Compile Options Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Options button inside the Settings panel.
  2. Inside the Directives group box choose the component for which the compile optimizations and report behavior should be changed in the Use the compiler options below for the Component combo box.
  3. Choose the desired warning level in the Warning Level combo box.
    The following choices are available:

    Severity

    Description

    No Warnings

    Does not list any warnings at all.

    Only severe

    Shows only the most severe warnings.

    Less severe

    Shows all warnings starting from a quite low severity.

    Almost all

    Shows all warnings but not informational messages.
    Mark the Treat all warnings as errors check box if a compilation should be considered as failed even when only warnings were issued.
  4. Specify how the generated code should be optimized by choosing one of the following options:

    Option

    Description

    None

    Disables all optimizations.

    Default

    Performs a default optimization on the compiled code. It depends on the compiler used what type of performance options are implemented for this option.

    Maximize Speed

    Optimizes for maximum speed.
    This is the recommended option.

    Minimize Size

    Minimizes the size of the compiled code.
  5. Push the OK button.

Prerequisites:
The privilege 'Changing the database compile options' must be enabled to configure the compile options.
