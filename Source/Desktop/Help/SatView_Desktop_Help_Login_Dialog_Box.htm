Log-in Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ supports a dialog based log-in interface for interactive users.
The Log-in Dialog Box appears automatically whenever a user authentication is required. Any user trying to log-in must have a valid user account.

How do I...

Perform a Log-in

  1. Choose the spacecraft to be monitored in the Spacecraft combo box.
  2. Select the user name in the User combo box.
  3. Select one of the databases available for the selected spacecraft and user in the Database combo box.
  4. Check the Online radio button if real-time telemetry data is going to be processed or Offline if only historical telemetry data is required.
    Note:
    Selecting the offline option logs-out from the telemetry data provider device but does not stop an (existing) physical connection to a remote network. In order to disconnect from a remote network select the 'No remote network or disconnect' option in the Server combo box.
  5. Enter the password into the Password edit field.
  6. Push the OK button.

Perform a remote Log-in

  1. Follow the steps above related to a successful (local) log-in.
    Note:
    A user account on the local workstation as well as on the remote server is required.
  2. Select the server in the Server combo box inside the Remote Network group box.
    Note:
    The Dial-up check box indicates if the selected connection is using a modem i.e. the phone line. Otherwise it is configured as a VPN connection.
  3. Enter the user name and password of the remote account by typing that information into the corresponding edit fields.
  4. Push the OK button.

Specify remote Log-in Options

  1. Push the Options... button in the Log-in Dialog Box. The Log-in Options Dialog Box will appear.
  2. Mark the Try to reconnect automatically if the connection is lost check box if a connection should be re-established after it has been broken.
  3. Mark the Try every n minutes check box to force several connection retries with an interval of n minutes and enter n into the corresponding spin box.
    Note:
    Omitting a limitation of retries and specifying a retry interval at the same time results in an indefinite loop of attempts for non-scheduled log-ins.
  4. Push the OK button.

Schedule automatic remote Log-ins

  1. Push the Options... button in the Log-in Dialog Box. The Log-in Options Dialog Box will appear.
  2. Push the Add... button in order to add a scheduled log-in. This opens the Log-in Schedule Dialog Box.
  3. Add the start time and stop time of the period at which the log-in should take place into the From t1 to t2 edit fields.
    Note:
    Include a margin of 1 to 2 minutes for dialing-in at the remote site when specifying the time range. When using the callback option it might even take slightly longer. The time ranges of multiple log-ins may overlap.
  4. Choose the server to log-in at in the Server combo box.
    Note:
    For convenience reasons the server name of the previously entered log-in is automatically selected.
  5. Specify the log-in credentials by entering a valid user name and password into the User and Password edit fields.
    Note:
    For convenience reasons the user name and password of the previous log-in is automatically filled-in.
  6. Push the OK button.
  7. Repeat the previous five steps if more than one scheduled log-in has to be inserted.
    A scheduled log-in can be deleted any time by selecting it in the List list box and by pushing the Remove button. The Remove All button clears all scheduled log-ins.
  8. Push the OK button.
