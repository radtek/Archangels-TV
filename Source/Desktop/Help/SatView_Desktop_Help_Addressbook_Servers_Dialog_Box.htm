Server Address Book Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ maintains a server address book containing relevant information about all available servers and is used mainly for log-in purposes when the telemetry data is received through the Remote Access Service (RAS).

How do I...

Add a new Server Address

  1. Open the Server Address Book Dialog Box if it is not open yet:
    Push the Servers button inside the Address Books panel of the Tools ribbon bar category.
  2. Choose the computer those server address book needs to be updated by selecting it in the Computer combo box or by typing its name into the corresponding edit field and pushing the Connect button.
    Note:
    The 'Changing the contents of a remote server address book' user privilege must be enabled in order to change the contents of a server address book on another computer.
  3. Enter the name of the server to be added in the Server edit field.
  4. Choose the communication device to be used in order to reach the server by selecting it in the Device combo box.
    Note:
    When updating a remote server address book it might be that no devices are available for selection.
  5. Specify the desired type of connection by marking the appropriate check box:

    Access

    Description

    VPN

    This option establishes a Virtual Private Network connection to the specified server.
    Note:
    The IP address (or DNS name) of the server has to be supplied (instead of a phone number).

    Phone

    The phone line (modem) should be used to connect to the server.
  6. Enter the dial-up number (or IP address for a VPN connection) into the Numbers edit field.
    Push the Add button in order to add it to the list of numbers (resp. addresses) associated with this server. Repeat this step as many times as necessary.
    In order to delete a number or IP address, select it first in the Numbers combo box and then push the Remove button.
    Note:
    The specified numbers must include, if necessary, extra characters to get an external line.
  7. Mark the Use callback mechanism check box if the server should call back and type the number to be called back into the attached edit field. This is useful to avoid extensive costs on the side of the client.
    Note:
    This option applies to phone-type connections only.
  8. Mark the Try all available numbers check box in order to force all numbers (or IP addresses) to be tried until one of them provides a successful connection.
  9. Mark the Use modem speaker or Use software compression check box if one of these options is required.
    Note:
    This option applies to phone-type connections only.
  10. Push the OK button.

Prerequisites:
The privilege 'Changing the contents of the (a remote) server address book' must be enabled.

Modify a Server Address

  1. Open the Server Address Book Dialog Box if it is not open yet:
    Push the Servers button inside the Address Books panel of the Tools ribbon bar category.
  2. Choose the computer those server address book needs to be updated by selecting it in the Computer combo box or by typing its name into the corresponding edit field and pushing the Connect button.
    Note:
    The 'Changing the contents of a remote server address book' user privilege must be enabled in order to change the contents of a server address book on another computer.
  3. Select the name of the server those details need to be modified in the Server combo box.
  4. Perform the required changes:
    • Change the communication device to be used in order to reach the server by selecting it in the Device combo box.
      Note:
      When updating a remote server address book it might be that no devices are available for selection.
    • Alter the way how the connection to the server should be established by marking either the VPN or Phone check box.
    • Change the dial-up numbers (or IP addresses for a VPN connection), if required, inside the Numbers list box i.e. push either the Add button in order to add one to the list of numbers or use the Remove button to delete an existing one.
    • Mark the Use callback mechanism check box if the server should call back and type called back number into the attached edit field. This is useful to avoid extensive costs on the side of the client.
      Note:
      This option applies to phone-type connections only.
    • Mark the Try all available numbers check box in order to force all numbers (or IP addresses) to be tried until one of them provides a successful connection.
    • Mark the Use modem speaker or Use software compression check box if one of these options is required.
      Note:
      This option applies to phone-type connections only.
    Note:
    For more details about all the configuration options read how to add a server address (see above).
  5. Push the OK button.

Prerequisites:
The privilege 'Changing the contents of the (a remote) server address book' must be enabled.

Delete a Server Address

  1. Open the Server Address Book Dialog Box if it is not open yet:
    Push the Servers button inside the Address Books panel of the Tools ribbon bar category.
  2. Choose the computer those server address book needs to be updated by selecting it in the Computer combo box or by typing its name into the corresponding edit field and pushing the Connect button.
    Note:
    The 'Changing the contents of a remote server address book' user privilege must be enabled in order to change the contents of a server address book on another computer.
  3. Select the name of the server to be deleted in the Server combo box.
  4. Push the Remove button. Use the Remove All button in order to delete all entries in the address book.
  5. Push the OK button.

Prerequisites:
The privilege 'Changing the contents of the (a remote) server address book' must be enabled.

Print Server Addresses

  1. Select the Print Server Address Book... item on the application button's menu.
  2. Fill-in all information into the Print Dialog Box.
    Note:
    Use the Page Setup Dialog Box in order to change the current print page settings.
  3. Push the OK button.
