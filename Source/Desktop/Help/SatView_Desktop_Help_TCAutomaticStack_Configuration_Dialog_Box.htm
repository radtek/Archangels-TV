TC Automatic Stack Settings Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

Many aspects of the TC Automatic Stack are configurable. Layout, operational aspects, filtering & logging options and the print profile can be modified as required. This command source does also have an in-built interface to a Mission Planning System (MPS).

How do I...

Modify the Layout

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Layout page.
  3. Specifying the Appearance:
    Choose the pane for which the layout should be modified inside the For Pane combo box.

    Creating a new Column

    1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
    2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
    3. Push the Insert Before or Insert After button.
      Note:
      The No. column must always be the left most one and it cannot be deleted.

    Deleting a Column

    1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
    2. Push the Remove button.
      Note:
      The No. column cannot be deleted.

    Specifying the Background Grid & Color

    • Mark the Grid check box in order to enable a background grid and select the desired color inside the associated color combo box.

    Changing the Fonts

    1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the telecommand stack.
    2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.
    Controlling the Operating Flags:

    Tool-tips

    A small tool-tip containing telecommand related details can be displayed on request. When this option is enabled and the cursor is moved over a telecommand inside the stack, a small hand occurs indicating that a tool-tip is available for that part of the telecommand. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

    • Mark the Tooltips check box in order to enable this feature.
  4. Push the OK button.

Change the Operational Aspects

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Operations page.

  3. Specifying the Operational Modes & Checks

    1. Choose the appropriate telecommand release modes:

      Mode

      Description

      Master Manual Mode

      Prevents all command sources from issuing any telecommands except for this command source.

      Release Wait Mode

      Forces this command source to apply the release wait mode which allows the release of telecommands within the specified (default) time window. If this mode is not enabled, a telecommand release is considered as failed as soon as the release time is encountered.

      Automatic Rejection Mode

      This mode automatically removes a telecommand that failed to be released from this command source.

      Disabled

      Prevents the issue of telecommands from this command source.
    2. Choose the appropriate telecommand release constraints:

      Constraint

      Description

      Override Pre-Transmission Validation (PTV)

      Forces any pre-transmission validation checks to result in a positive ('Go') result even when some or all conditions are not met.

      Override Pre-Execution Verification (PEV)

      Forces any pre-execution verification checks to result in a positive ('Go') result.

      Override Execution Verification (CEV)

      Forces any execution verification checks to result in a successful ('Passed') result.

      Disable PTV

      Does not perform any pre-transmission validation checks (implicit 'Go').

      Disable PEV

      Does not perform any pre-execution verification checks (implicit 'Go').

      Disable CEV

      Does not perform any execution verification checks.

      Note:
      The above checks apply to this command source only. The telecommands dispatched may have different checks enabled.

      Adjust some additional telecommand release constraints and checks:

      Constraint

      Description

      Enable local Interlocks

      Allows the interlocks local to this command source to control the release of telecommands. If this option is not selected the interlocks have no affect.

      Continuous checks for the first n telecommands

      Constantly checks and monitors the pre-transmission and pre-execution state of the n first telecommands.
      Note:
      Checking the pre-transmission and pre-execution states can be a time consuming task. A number n up to 100 is sufficient in most cases.
    3. Specify miscellaneous other configuration options:

      Telecommand Arguments:

      Option

      Description

      List limited to n Arguments

      When this option is enabled only the first n arguments will be displayed on the telecommand stack.

      Show fixed ones too

      Lists the fixed (unchangeable) arguments as well.

      Various Time-outs:

      Type

      Description

      Default release time window

      Specifies the default telecommand release time window (in seconds) i.e. the interval proposed when a release time window is requested for a telecommand.

      Dispatch time window

      Defines the time interval (in seconds) between the moment where a telecommand is armed and the time it has to be dispatched. Otherwise it will be switched back to an on-stack status.

      Exit Behavior:

      Behavior

      Description

      Always allow exit

      Shutting down this command source is possible when this option is enabled even when there are dispatched telecommands on the stack.
      All telecommands are restored with their current status when operations on this command source are resumed.
      Note:
      Specified release time constraints remain intact and can cause a failure when they cannot be met any more after the shut-down period.

      Undispatch contents on exit

      All dispatched telecommands are automatically converted back to an on-stack status when this command source is shut down.

    Specifying the Alert Behavior

    1. Open the TC Automatic Stack Alert Options Dialog Box if it is not open yet:
      Push the Alerts... button.
    2. Select the type of incident that should be alerted inside the Type combo box:

      Incident

      Description

      Telecommand release failures

      Covers all situations where one or several telecommands cannot be released due to any reason.

      Interlock failures

      Describes situations where a local interlock switches to a failed state.
    3. Inside the Action group box select the desired audition through the Audition combo box that should be performed when the above selected incident occurs. It will turn up as an event in the Global Eventbox with the color specified in the associated Color combo box. Mark the Blinking & audible check box if the event should also blink (incl. audible sound if chosen) and require an acknowledgment.
    4. Add all notifications to be performed (if any) when an incident of the selected type occurs by choosing them first in the Notifications combo box and then by pressing the Add button.
      Use the Remove button to delete the selected notification profile or Remove All to delete all of them at once.
    5. Push the OK button.

    Configuring User Permissions

    Sometimes it might be necessary to extend the rights for all those users that do not have the 'Performing privileged tasks on the automatic telecommand stack' privilege.

    1. Open the TC Automatic Stack User Permissions Dialog Box if it is not open yet:
      Push the button with a key symbol as label.
    2. Choose the additional privileges to be granted:

      Privilege

      Description

      Change the telecommand PTV, PEV and CEV checks

      Grants the right to modify the telecommand release validation and (pre-)execution verification checks for selected telecommands.

      Change the telecommand grouping or blocking flags

      Allows the grouping of telecommands as well as the encoding of multiple telecommands into a single telecommand packet.

      Change the telecommand release time

      Allows the change of the telecommand release time.

      Insert or remove interlocks

      Allows the creation, modification and deletion of interlocks.

      Change the telecommand on-board execution time

      Allows the modification of the scheduled on-board execution time of a telecommand.

      Modify the telecommand parameter values

      Allows the change of telecommand parameter values.

      Use templates

      Allows the creation, modification and deletion of templates.
    3. Push the OK button.

    Note:
    Only users owning the privilege 'Performing privileged tasks on the automatic telecommand stack' can grant any of the above privileges to those without.
  4. Push the OK button.

Changing the Filtering Options

The Historical Panes of the Normal Stack and the Time-Tagged Stack can apply filters with respect to various criteria in order to restrict the telecommands displayed.

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Filtering page.
  3. Choose the desired filtering criteria:

    Filter Option

    Description

    Telecommand

    Applies a filter with respect to properties of telecommands:

    Property

    Description

    Name

    Filters all telecommands with a specified name i.e. all telecommands with a different name are not displayed.

    Sequence

    Filters all telecommand sequences with a specified name.

    Type

    Filters all telecommands with either an Immediate, Time-Tagged, 1st Step or 2nd Step type.

    Sub-system

    Filters all telecommands related to a certain sub-system:

    1. Select the desired sub-system in the Sub-system combo box.
    2. Push the < button. This adds it to the list of sub-systems.
      Repeat the previous steps to add more or use the > button to remove a selected entry.

    Verification Status

    Filters all telecommands those stage verification check ended in a certain result. The following stage verification checks can be compared with these predefined results:

    Stage

    Result

    Transmission
    Acceptance
    Start
    Progress
    Completion

    Passed
    Failed
    Unknown
    Unverified
    1. Select the stage verification check in the Stage combo box.
    2. Mark the State check box and choose the required check result in the associated combo box.
      Repeat the previous steps if more that one stage check should be included in the filter.

    Note:
    Telecommands not implementing a stage check that is included in the filter are not listed.

    Limited to maximal n entries

    Limits the number of telecommands listed to n entries.

    Note:
    Choosing multiple filtering options results in combining them with a logical 'AND'.

  4. Choose the desired sorting order:

    Sorting Option

    Description

    By Transmission Time

    The telecommands are listed according to their transmission time.

    By Execution Time

    The telecommands are listed according to their on-board execution time. If there is no on-board execution time attached to the telecommand the transmission time is used instead.

    Show the latest entry as first one

    Determines if the latest telecommands are inserted at the top or appended at the bottom.
  5. Push the OK button.

Note:
The filtering options apply to the Historical Panes only.

Configuring the Mission Planning Interface

Telecommands generated by a Mission Planning System as well as data from a Flight Dynamics System can be imported manually or automatically. Import filters for third-party software can be implemented on request.

Mission Planning System:

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Planning page.
  3. Inside the Mission Planning System group box choose the workstation hosting the MPS in the Import from combo box. This will be the first choice for importing the generated schedules.
  4. Mark the Backup check box and select an alternative backup source inside the associated combo box if a backup is desired.
  5. Inside the Options group box choose one of the following import directives:

    Directive

    Description

    Manual imports only

    This directive prevents any automatic import of telecommand schedules.

    Automatically import n hours ahead

    Allows manual imports as well as automatic ones. Any telecommands with a release time later than n hours ahead are discarded.
  6. Choose the appropriate report behavior in case any problems arise during the import of data:

    Behavior

    Description

    Nothing

    Nothing at all is reported.

    Everything

    All kind of messages including informational ones are reported:

    Type

    Description

    Audition

    Generates an audible alarm.
    Mark the Audition check box and select the desired audition in the associated combo box.

    Notification

    Informs with a notification (e.g. e-mail, phone call etc.).
    Mark the Notification check box and select the desired notification in the associated combo box.

    Failures

    Only failures are reported:

    Type

    Description

    Audition

    Generates an audible alarm.
    Mark the Audition check box and select the desired audition in the associated combo box.

    Notification

    Informs with a notification (e.g. e-mail, phone call etc.).
    Mark the Notification check box and select the desired notification in the associated combo box.
  7. Push the OK button.

Flight Dynamics System:

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Planning page.
  3. Inside the Flight Dynamics System group box choose the workstation hosting the FDS in the Import from combo box. This will be the first choice for importing the generated data.
  4. Mark the Backup check box and select an alternative backup source inside the associated combo box if a backup is desired.
  5. Inside the Options group box choose how the data should be synchronized:

    Behavior

    Description

    Perform automatic synchronization

    Enables an automatic synchronization with the FDS as required and defined by internal directives.

    Allow manual synchronization

    Restricts any synchronization to a manual interface.
  6. Push the OK button.

Note:
This interface is not yet operational and will be complemented by additional features.

Change the Logging Options

Every change with respect to the operational configuration of this command source can be logged and reviewed at a later time. This allows the reconstruction of any configuration changes.

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Logging page.

  3. Configuring the Options

    1. Push the Options... button. The TC Automatic Stack Logging Options Dialog Box will appear.
    2. Type the name for the log file into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
      Note:
      The file name can be identical to those of the other (remote) command sources. This may be interesting if only one single log file should be generated.
    3. Check one of the available file policy options:

      Option

      Description

      Overwrite logs as needed

      Enables the cyclic overwrite mode.
      Note:
      As soon as the log file reaches the specified size or no more space is available on the storage media new logs overwrite the oldest ones.

      Overwrite logs older than n days

      Enables the cyclic overwrite mode.
      Note:
      Logs are overwritten by new ones as soon as they are older than n days. This can happen even before the specified file size is reached.

      Do not overwrite any logs

      Disables the cyclic overwrite mode.
      Note:
      Stops logging when either the specified size is reached or no more space is available on the storage media.
      Optional:
      • Mark the Disabled check box in order to prevent the log file from being updated.
    4. Check one of the available file size options:

      Option

      Description

      Unlimited

      Archives logs as long as space is available on the storage media.

      Limited to n KBytes

      Limits the file size to n KB.
      Optional:
      • Mark the Reset check box in order to reset the log file.
    5. Push the Retrievals... button which opens the TC Automatic Stack Log Retrieval Options Dialog Box and type the name of the log file to be used for retrievals into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
      Push the OK button.
      Note:
      The file name is usually identical to the one specified for logging but can also be identical to those of the other (remote) command sources. This may be interesting if only one single file is used for the logging of all command sources.
    6. Push the OK button.

    Viewing Real-time Log Entries

    1. Check the Realtime radio button.
    2. Mark the Reload the last n entries check box, enter the number n into the associated edit field and push the Retrieve button in order to immediately retrieve the last n log entries.
      Note:
      By enabling this option the last n log entries will always be listed whenever the dialog box is opened.

    Retrieving Log Entries

    1. Check the Historical radio button and enter the start time for the retrieval into the associated edit field.
    2. Specify the stop condition by checking either:
      • Stop at t and enter the desired stop time t
      • Limit to n entries and enter the number n of entries to retrieve
      • Retrieve up to the end of the log file radio button
    3. Push the Retrieve button.

    Redirecting Log Entries

    • Mark the Show Event check box in order to redirect each log entry as an event to the Global Eventbox.
  4. Push the OK button.

Modify the Print Profile

  1. Open the TC Automatic Stack Settings Dialog Box if it is not open yet:
    Activate the TC Automatic Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Printing page.
  3. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
  4. Inside the Options group box choose those telecommands that should be printed:

    Stack

    Description

    Normal

    Prints telecommands from the Normal Stack i.e. telecommands that have an immediate on-board execution time.

    Time-Tagged

    Prints telecommands from the Time-Tagged Stack i.e. telecommands that have a scheduled on-board execution time.

    Status

    Description

    On Stack

    Prints all telecommands currently in an on-stack status.

    Dispatched

    Prints all telecommands currently in a dispatched status.

    Released

    Prints all telecommands that were released, sent to the ground station or uplinked.

    Not Released

    Prints all telecommands that were not released due to failed pre-transmission verification checks, encoding failures or denied release authorization.

    Failed Release

    Prints all telecommands that failed to be released. This option also includes failed send and uplink attempts.

    Range

    Description

    No. from n1 to n2

    Prints all telecommands between number n1 and n2.

    Release Time from t1 to t2

    Prints all telecommands between time t1 and t2.

    Note:
    If several filter options are selected they are combined through a logical 'AND'.

  5. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
  6. Push the OK button.

Prerequisites:
The privileges 'Changing the contents of the automatic telecommand stack' or 'Performing privileged tasks on the automatic telecommand stack' must be enabled.
