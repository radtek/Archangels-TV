Account Databases Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

User accounts also contain the list of databases to which a user has access to. This dialog box allows the modification of the account database access list.

How do I...

Add a Database

  1. Open the Account Databases Dialog Box if it is not open yet:
    Push the Users button inside the Accounts panel of the System ribbon bar category and use the Databases... button inside the Privileges group box of the dialog box.
  2. Select the host workstation of the database to which the current user should get access to inside the Server combo box.
    Note:
    It is also possible to enter the name of a server manually. By pushing the Connect button all databases available on that workstation will get available for selection.
  3. Choose the desired database in the Databases combo box.
  4. Push the Add button in order to add the selected database to the list of accessible databases.
    Repeat the last two steps until all required databases are added to the list.
  5. Push the OK button.

Remove a Database

  1. Open the Account Databases Dialog Box if it is not open yet:
    Push the Users button inside the Accounts panel of the System ribbon bar category and use the Databases... button inside the Privileges group box of the dialog box.
  2. Choose the host workstation of the database to which the current user should loose access to inside the Server combo box.
    Note:
    Enter the name of the desired remote workstation manually if it is not listed and push the Connect button.
  3. Select the database to be removed from the account access list inside the Allowed Databases list box.
  4. Push the Remove button. Use the Remove All button in order to empty the database access list.
  5. Push the OK button.

Prerequisites:
The privilege 'Changing the allowed databases of the own account' must be enabled in order to modify the database access list of the account if 'Creating and modifying (remote) user accounts' is not enabled.
