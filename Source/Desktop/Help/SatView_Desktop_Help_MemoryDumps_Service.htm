Memory Dumps Service

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ is able to acquire and process telemetry data dumped from the spacecraft through the Memory Dumps Service. The collected data can (optionally) be further processed by user-defined automation scripts, archived or sent to other users via FTP transfer. The Dumps type of the Hexadecimal Display (HEX) is the first choice to view such memory dumps. Each memory dump to be processed must be registered by an associated profile. The Memory Dump Service is a light-weight solution compared to the TC On-board Memory Images Display which is used to reflect the complete on-board memory status of a spacecraft.

How do I...

Configure the Memory Dump Service

  1. Open the Memory Dumps Service Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Memory Dumps panel of the Telemetry ribbon bar category.
  2. Select the Enable Memory Dumps Service check box in order to enable the service.
  3. Choose the appropriate options in the Availability group box:

    Option

    Description

    Unlimited

    The Memory Dumps Service is available all the time.

    Daily between t1 and t2

    Restricts the availability of the Memory Dumps Service to a daily period between t1 and t2.

    Working Days only

    The Memory Dumps Service is available in the specified way between Monday and Friday only.
    Note:
    This option can be combined with one of the above.
    Note:
    Mark the Audit check box if an event message should be issued whenever the availability of the service changes.
  4. Push the OK button.

Prerequisites:
The privilege 'Changing the memory dumps service settings' must be enabled in order to modify the settings of the service.

Create a new Memory Dump Profile

Memory dump profiles define the properties of those dumps that need to be processed.

  1. Open the Memory Dumps Service Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Memory Dumps panel of the Telemetry ribbon bar category.
  2. Inside the Dump Properties group box enter the name of the memory dump profile to be created into the Name edit field.
  3. Select the source of the memory dump to be processed in the Source combo box. The source specifies the subsystem on-board of the spacecraft that generates the dump.
    Note:
    The number of sub-systems able to generate dumps depends on the spacecraft.
  4. Enter the address (in hexadecimal numbers) of the memory dump into the Address edit fields. Together with the source information the address is used to identify the dump.
  5. Mark the Audit check box if a message should be generated whenever a memory dump that fits the profile characteristics was processed.
  6. Select the format type of the memory dump to be generated in the Type combo box.
    Note:
    The available format types depend on the spacecraft and the operational requirements.
  7. On demand, downloaded memory dumps can be archived and sent via FTP to any specified user:

    Archiving a Memory Dump

    1. Select the Archiving page (at the bottom).
    2. Specify the filename to be used when the memory dump is archived. The filename must contain a valid file path and file title but no extension (e.g. 'path\title'). The Memory Dumps Service will add the time the memory dump was generated after the file title by using the following syntax: YYYYMMDD_hhmmss.
    3. Choose the formats of the memory dump to be generated by marking the appropriate Formats check boxes and filling in the file extensions to be added to the filenames by typing them into the Extensions edit fields. The syntax of the resulting memory dump filenames are: 'path/title_YYYYMMDD_hhmmss.ext'.
    4. Mark the Enable Archiving check box inside the Retentions group box in order to enable the archiving of the memory dump.
    5. If required, limit the number of memory dumps to be archived by selecting the Stop after n dumps check box and entering the number of dumps n into the associated spin box and/or select the Stop at date check box and specify the date after that no memory dumps should be archived any more.

    Notifying a Memory Dump

    1. Select the Notifications page (at the bottom).
    2. Specify the location where the memory dump should be sent to. The destination address must be a valid FTP site address in the format 'ftp://domain/directory' where 'domain' identifies a domain name or IP address and 'directory' an existing directory.
      Note:
      Multiple destination addresses may be specified, if required, each of them separated by a semicolon or comma.
    3. Enter the user name and password into the corresponding Username and Password edit fields.
      The following convention related to user authentication is summarized in the table below:

      Username

      Password

      Username sent to FTP Site

      Password sent to FTP Site

      Empty

      Empty

      Anonymous

      E-Mail Address

      Username

      Empty

      Username

      Empty

      Username

      Password

      Username

      Password
      Note:
      If multiple destination addresses are specified, the corresponding user names and passwords should be separated by a semicolon or comma. Entering fewer user names and passwords than available destination addresses associates an empty user name and password to those addresses.
    4. Specify a TCP/IP port number by marking the Use Port check box and entering a port number if a dedicated port is required for the FTP transfer.
    5. Select the memory dump formats to be transferred to the FTP site by marking the corresponding check boxes.
    6. Mark the Enable Notifications check box inside the Retentions group box in order to enable the FTP transfers of the memory dump.
    7. If required, limit the number of memory dumps to be transferred by selecting the Stop after n dumps check box and entering the number of dumps n into the associated spin box and/or select the Stop at date check box and specify the date after that no memory dumps should be sent to clients any more.

    Specifying the FTP Options

    1. Select the FTP Transfers page (at the bottom).
    2. Specify the action that should be taken whenever a problem during a FTP transfer occurs:

      Option

      Description

      Retry Never

      No action is taken when a FTP transfer was not successful.

      Retry Continuously

      Continuously retry to send the memory dump at an interval (in minutes) specified in the Interval spin box.

      Retry n times

      As above but restricted to n retries.
      Choose the following options whenever required:

      Option

      Description

      Reset queue

      Removes all memory dumps currently in the transmission queue (for example due to a previous failure). This prevents a transfer retry at a later time.

      Save on exit

      Saves the number of unsuccessful retries when shutting down. All memory dumps that could not be transferred up to that moment are stored and sent as specified at a later time.
    3. Specify the failure report options by selecting the appropriate radio buttons in the Audition group box:

      Option

      Description

      Report Nothing

      No failures are reported at all.

      Report Everything

      An event message is generated whether a successful FTP transfer occurs or a problem turns-up.

      Report Failures

      Only failures during a FTP transfer are reported.
  8. Push the OK button.

Prerequisites:
The privilege 'Changing the memory dumps service settings' must be enabled in order to create a new memory dump profile.

Modify a Memory Dump Profile

  1. Open the Memory Dumps Service Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Memory Dumps panel of the Telemetry ribbon bar category.
  2. Inside the Dump Properties group box select the name of the memory dump profile to be modified in the Name combo box.
  3. Perform the necessary changes (see 'Creating a new Memory Dump Profile' for more details) and then push the Modify button.
    Note:
    The previous two steps can be repeated as many times as memory dump profiles need to be modified.
  4. Push the OK button.

Prerequisites:
The privilege 'Changing the memory dumps service settings' must be enabled in order to modify a memory dump profile.

Delete a Memory Dump Profile

  1. Open the Memory Dumps Service Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Memory Dumps panel of the Telemetry ribbon bar category.
  2. Inside the Dump Properties group box select the name of the memory dump profile to be deleted in the Name combo box.
  3. Push the Remove button.
    Note:
    Repeat the previous steps as many times as memory dump profiles need to be deleted.
  4. Push the OK button.

Prerequisites:
The privilege 'Changing the memory dumps service settings' must be enabled in order to delete a memory dump profile.

Import Memory Dump Profiles

SatView™ supports import/export interfaces for memory dump profiles, allowing the synchronization of multiple workstations in an easy way.

  1. Open the Memory Dumps Import Dialog Box if it is not open yet:
    Push the Import... button inside the Memory Dumps panel of the Telemetry ribbon bar category.
  2. Enter the name of the file that contains all memory dump profiles to be imported into the Filename edit field.
  3. Push the Inspect button in order to list all profiles that are contained in the specified file.
    Note:
    The Inspect button is available only when a file name is specified. With an empty Filename edit field the Browse... button is visible instead.
  4. In the Directory combo box select a memory dump profiles that should be imported.
  5. Push the Add button. Repeat the previous two steps if more than one memory dump profile should be imported or use the Add All button in order to import all available ones.
    Use the Remove or Remove All button in order to remove one or several memory dump profiles from the importation list.
  6. Push the OK button.

Prerequisites:
The privilege 'Importing memory dump profiles' must be enabled in order to import memory dump profiles.

Export Memory Dump Profiles

SatView™ supports import/export interfaces for memory dump profiles, allowing the synchronization of multiple workstations in an easy way.

  1. Open the Memory Dumps Export Dialog Box if it is not open yet:
    Push the Export... button inside the Memory Dumps panel of the Telemetry ribbon bar category.
  2. In the Directory combo box select a memory dump profile that should be exported.
  3. Push the Add button. Repeat the previous steps if more than one memory dump profile should be exported or use the Add All button in order to export all available ones.
    Use the Remove or Remove All button in order to remove one or several memory dump profiles from the exportation list.
  4. Enter the name of the file that should contain all exported memory dump profiles into the Filename edit field.
    Use the Browse... button in order to search for an appropriate destination file if necessary.
  5. Push the OK button.

Prerequisites:
The privilege 'Exporting memory dump profiles' must be enabled in order to export memory dump profiles.

Explore Memory Dumps

The preferred way to view memory dumps is through Hexadecimal Displays (HEX). These displays also allow retrievals with respect to historical memory dumps.

  1. Open a Hexadecimal Display (HEX) by pushing the HEX button inside the Telemetry Displays panel of the Displays ribbon bar category.
  2. Choose the Dumps check box which specifies the display type required for this purpose.
  3. Select the memory dump profile corresponding to the dump to be displayed in the Display combo box.
  4. Choose the requested data mode by either checking the Realtime or History radio button.
    Note:
    Follow the instructions about how to retrieve with a HEX display when historical dumps are going to be viewed.
  5. Push the OK button.

Prerequisites:
The privilege 'Using hexadecimal display windows' must be enabled.
