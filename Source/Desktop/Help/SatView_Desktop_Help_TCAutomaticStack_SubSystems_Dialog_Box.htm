TC Automatic Stack Sub-systems Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Automatic Stack has the capability to enable/disable the release of telecommands targeting a certain sub-system on-board of the spacecraft either manually or automatically depending on telemetry events.

How do I...

Control the Release of Telecommands for a certain Sub-system:

By Telemetry Events

  1. Open the TC Automatic Stack Sub-systems Dialog Box if it is not open yet:
    Select the Sub-system Checks item on the application button's menu when the display is activated. Alternatively, push the Sub-system Checks on the display's toolbar.
  2. Select the on-board sub-system for which the release of telecommands should be allowed or denied inside the Sub-systems list box.
  3. Mark the Enabled check box and select the telemetry event enabling the release of telecommands in the upper combo box and/or choose the event disabling the release in the lower combo box.
    Note:
    If no telemetry event is attached to the enabled state, the release of telecommands is always allowed except for situations where the event assigned to the disabled state is signaled. In the other case where an event is attached to the enabled state only, the release of telecommands is allowed solely when this event is signaled.
  4. Push the OK button.

Prerequisites:
The privileges 'Changing the contents of the automatic telecommand stack' or 'Performing privileged tasks on the automatic telecommand stack' must be enabled to control the release of telecommands this way.

Manually

  1. Open the TC Automatic Stack Sub-systems Dialog Box if it is not open yet:
    Select the Sub-system Checks item on the application button's menu when the display is activated. Alternatively, push the Sub-system Checks on the display's toolbar.
  2. Select the on-board sub-system for which the release of telecommands should be allowed or denied inside the Sub-systems list box.
  3. Mark the Enabled check box to allow the release of telecommands or Disabled to deny it.
    Note:
    In order to force a constant setting (i.e. independent from any telemetry data) no events must be attached.
  4. Push the OK button.

Prerequisites:
The privileges 'Changing the contents of the automatic telecommand stack' or 'Performing privileged tasks on the automatic telecommand stack' must be enabled to control the release of telecommands this way.
