TC Manual Stack Save Telecommand Stacks Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Manual Stack can handle so called 'saved stacks' which are nothing else than predefined portions of the stack saved to a repository for later re-use.

How do I...

Add Telecommand Stacks to a Repository

  1. Select all those telecommands that should become part of the saved stack.
  2. Open the TC Manual Stack Save Telecommand Stacks Dialog Box if it is not open yet:
    Select the Save... item on the application button's menu when the display is activated. Alternatively, push the Save button on the display's toolbar.
  3. Enter a name for the selected portion of the stack into the Name edit field.
  4. Type a short description into the Description edit field summarizing the purpose of this saved stack. This simplifies future searches.
  5. Choose the appropriate storing options:

    Option

    Description

    Use relative release times

    Converts absolute telecommand release times into relative ones i.e. the release time of all telecommands will be converted to a delta time with respect to the first one. This greatly simplifies a later re-use.

    Use relative on-board execution times

    Converts absolute on-board execution times into relative ones with the delta execution time relative to the execution time (or release time if no execution time is available) of the first telecommand.
  6. Mark the Validated check box if no consistency check with respect to the database should be performed when this stack is loaded at a later time.
    Note:
    Leave this option disabled in order to make sure that all telecommands are checked against the database on the workstation loading the saved stack.
  7. Inside the Repository group box enter a file name for the repository to store the stack into the Filename edit field.
    Note:
    The specified file may exist already and contain other saved stacks. Push the Inspect button in order to examine its current contents.
  8. Push the OK button.

Prerequisites:
The privileges 'Changing the contents of the manual telecommand stack' or 'Performing privileged tasks on the manual telecommand stack' must be enabled to create, modify or delete repositories.

Remove Telecommand Stacks from a Repository

  1. Open the TC Manual Stack Save Telecommand Stacks Dialog Box if it is not open yet:
    Select the Save... item on the application button's menu when the display is activated. Alternatively, push the Save button on the display's toolbar.
  2. Inside the Repository group box enter a file name for the repository into the Filename edit field.
    Use the Browse... button to search for an existing file.
  3. Push the Inspect button. This lists the contents of the repository in the lower list box.
  4. Select the saved stack to be removed from the repository and then push the > button or use the >> button to empty it.
  5. Push the OK button.

Prerequisites:
The privileges 'Changing the contents of the manual telecommand stack' or 'Performing privileged tasks on the manual telecommand stack' must be enabled to create, modify or delete repositories.
