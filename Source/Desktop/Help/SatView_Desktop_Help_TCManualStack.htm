TC Manual Stack

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC Manual Stack is the major command source of SatView™. It is designed to manually load telecommands or telecommand sequences from the database and to eventually edit them before release. All telecommands need to be dispatched before they can be released by either a manual or automatic dispatch condition.
The TC Manual Stack supports interlocks, grouping and blocking links as well as the automatic rejection mode and release wait mode. Control over all operational aspects like Pre-transmission Validation (PTV), Pre-execution Verification (PEV) or Command Execution Verification (CEV) checking, running modes etc. is provided as well together with the capability to optionally log all activities concerning the configuration changes of the TC Manual Stack.

How do I...

Open the TC Manual Stack

  1. Push the Manual TC Stack toggle-button inside the Command Sources panel of the Telecommand ribbon bar category or, alternatively, open the Windows button's menu inside the Window panel of the System ribbon bar category and select the TC Manual Stack item.
  2. Optionally configure the TC Manual Stack.

Note:
The TC Manual Stack does not show released telecommands. Use the TC History Display for that purpose.

Configure the Settings

Many aspects of the TC Manual Stack are configurable. Layout, operational aspects, logging options and the print profile can be modified as required.

  1. Open the TC Manual Stack Settings Dialog Box if it is not open yet:
    Activate the TC Manual Stack and select Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.

    2. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      3. Push the Insert Before or Insert After button.
        Note:
        The No. column must always be the left most one and it cannot be deleted.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        The No. column cannot be deleted.

      Specifying the Background Grid & Color

      • Mark the Grid check box in order to enable a background grid and select the desired color inside the associated color combo box.

      Changing the Fonts

      1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the telecommand stack.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Tool-tips

      A small tool-tip containing telecommand related details can be displayed on request. When this option is enabled and the cursor is moved over a telecommand inside the stack, a small hand occurs indicating that a tool-tip is available for that part of the telecommand. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Changing the Operational Aspects

    1. Select the Operations page.

    2. Specifying the Operational Modes & Checks

      1. Choose the appropriate telecommand release modes:

        Mode

        Description

        Master Manual Mode

        Prevents all command sources from issuing any telecommands except for this command source.

        Release Wait Mode

        Forces this command source to apply the release wait mode which allows the release of telecommands within the specified (default) time window. If this mode is not enabled, a telecommand release is considered as failed as soon as the release time is encountered.

        Automatic Rejection Mode

        This mode automatically removes a telecommand that failed to be released from this command source.

        Disabled

        Prevents the issue of telecommands from this command source.
      2. Choose the appropriate telecommand release constraints:

        Constraint

        Description

        Override Pre-Transmission Validation (PTV)

        Forces any pre-transmission validation checks to result in a positive ('Go') result even when some or all conditions are not met.

        Override Pre-Execution Verification (PEV)

        Forces any pre-execution verification checks to result in a positive ('Go') result.

        Override Execution Verification (CEV)

        Forces any execution verification checks to result in a successful ('Passed') result.

        Disable PTV

        Does not perform any pre-transmission validation checks (implicit 'Go').

        Disable PEV

        Does not perform any pre-execution verification checks (implicit 'Go').

        Disable CEV

        Does not perform any execution verification checks.

        Note:
        The above checks apply to this command source only. The telecommands dispatched may have different checks enabled.

        Adjust some additional telecommand release constraints and checks:

        Constraint

        Description

        Enable local Interlocks

        Allows the interlocks local to this command source to control the release of telecommands. If this option is not selected the interlocks have no affect.

        Continuous checks for the first n telecommands

        Constantly checks and monitors the pre-transmission and pre-execution state of the n first telecommands.
        Note:
        Checking the pre-transmission and pre-execution states can be a time consuming task. A number n up to 100 is sufficient in most cases.
      3. Specify miscellaneous other configuration options:

        Telecommand Arguments:

        Option

        Description

        List limited to n Arguments

        When this option is enabled only the first n arguments will be displayed on the telecommand stack.

        Show fixed ones too

        Lists the fixed (unchangeable) arguments as well.

        Various Time-outs:

        Type

        Description

        Default release time window

        Specifies the default telecommand release time window (in seconds) i.e. the interval proposed when a release time window is requested for a telecommand.

        Dispatch time window

        Defines the time interval (in seconds) between the moment where a telecommand is armed and the time it has to be dispatched. Otherwise it will be switched back to an on-stack status.

        Exit Behavior:

        Behavior

        Description

        Always allow exit

        Shutting down this command source is possible when this option is enabled even when there are dispatched telecommands on the stack.
        All telecommands are restored with their current status when operations on this command source are resumed.
        Note:
        Specified release time constraints remain intact and can cause a failure when they cannot be met any more after the shut-down period.

        Undispatch contents on exit

        All dispatched telecommands are automatically converted back to an on-stack status when this command source is shut down.

      Specifying the Alert Behavior

      1. Open the TC Manual Stack Alert Options Dialog Box if it is not open yet:
        Push the Alerts... button.
      2. Select the type of incident that should be alerted inside the Type combo box:

        Incident

        Description

        Telecommand release failures

        Covers all situations where one or several telecommands cannot be released due to any reason.

        Interlock failures

        Describes situations where a local interlock switches to a failed state.
      3. Inside the Action group box select the desired audition through the Audition combo box that should be performed when the above selected incident occurs. It will turn up as an event in the Global Eventbox with the color specified in the associated Color combo box. Mark the Blinking & audible check box if the event should also blink (incl. audible sound if chosen) and require an acknowledgment.
      4. Add all notifications to be performed (if any) when an incident of the selected type occurs by choosing them first in the Notifications combo box and then by pressing the Add button.
        Use the Remove button to delete the selected notification profile or Remove All to delete all of them at once.
      5. Push the OK button.

    Changing the Logging Options

    Every change with respect to the operational configuration of this command source can be logged and reviewed at a later time. This allows the reconstruction of any configuration changes.

    1. Select the Logging page.

    2. Configuring the Options

      1. Push the Options... button. The TC Manual Stack Logging Options Dialog Box will appear.
      2. Type the name for the log file into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
        Note:
        The file name can be identical to those of the other (remote) command sources. This may be interesting if only one single log file should be generated.
      3. Check one of the available file policy options:

        Option

        Description

        Overwrite logs as needed

        Enables the cyclic overwrite mode.
        Note:
        As soon as the log file reaches the specified size or no more space is available on the storage media new logs overwrite the oldest ones.

        Overwrite logs older than n days

        Enables the cyclic overwrite mode.
        Note:
        Logs are overwritten by new ones as soon as they are older than n days. This can happen even before the specified file size is reached.

        Do not overwrite any logs

        Disables the cyclic overwrite mode.
        Note:
        Stops logging when either the specified size is reached or no more space is available on the storage media.
        Optional:
        • Mark the Disabled check box in order to prevent the log file from being updated.
      4. Check one of the available file size options:

        Option

        Description

        Unlimited

        Archives logs as long as space is available on the storage media.

        Limited to n KBytes

        Limits the file size to n KB.
        Optional:
        • Mark the Reset check box in order to reset the log file.
      5. Push the Retrievals... button which opens the TC Manual Stack Log Retrieval Options Dialog Box and type the name of the log file to be used for retrievals into the Filename edit field. Use the Browse... button in order to locate the file if necessary.
        Push the OK button.
        Note:
        The file name is usually identical to the one specified for logging but can also be identical to those of the other (remote) command sources. This may be interesting if only one single file is used for the logging of all command sources.
      6. Push the OK button.

      Viewing Real-time Log Entries

      1. Check the Realtime radio button.
      2. Mark the Reload the last n entries check box, enter the number n into the associated edit field and push the Retrieve button in order to immediately retrieve the last n log entries.
        Note:
        By enabling this option the last n log entries will always be listed whenever the dialog box is opened.

      Retrieving Log Entries

      1. Check the Historical radio button and enter the start time for the retrieval into the associated edit field.
      2. Specify the stop condition by checking either:
        • Stop at t and enter the desired stop time t
        • Limit to n entries and enter the number n of entries to retrieve
        • Retrieve up to the end of the log file radio button
      3. Push the Retrieve button.

      Redirecting Log Entries

      • Mark the Show Event check box in order to redirect each log entry as an event to the Global Eventbox.

    Modifying the Print Profile

    1. Select the Printing page.
    2. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
    3. Inside the Options group box choose those telecommands that should be printed:

      Status

      Description

      On Stack

      Prints all telecommands currently in an on-stack status.

      Dispatched

      Prints all telecommands currently in a dispatched status.

      Not Released

      Prints all telecommands that were not released due to failed pre-transmission verification checks, encoding failures or denied release authorization.

      Failed Release

      Prints all telecommands that failed to be released.

      Range

      Description

      No. from n1 to n2

      Prints all telecommands between number n1 and n2.

      Release Time from t1 to t2

      Prints all telecommands between time t1 and t2.

      Note:
      If several filter options are selected they are combined through a logical 'AND'.

    4. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
  3. Push the OK button.

    Command Source

    Command sources are the location where telecommands are usually prepared for release. They have multiple commanding states, modifiable telecommand release modes & constraints and provide many additional features in order to guarantee seamless operations.

    • Changing the Commanding State & Mode

      1. Open the TC Manual Stack Settings Dialog Box if it is not open yet:
        Activate the TC Manual Stack and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
      2. Select the Operations page.
      3. This command source can operate in three different commanding states:

        Commanding State

        Description

        COMMANDING

        The command source is in a state allowing the dispatch of telecommands and any other manipulation on the stack (except undispatching telecommands).
        Select the Start/Resume item on the application button's menu when the display is activated. Alternatively, push the Start/Resume button on the display's toolbar.

        SUSPENDED

        The command source is in a state preventing the dispatch of telecommands but allowing any other manipulation on the stack.
        Select the Suspend item on the application button's menu when the display is activated. Alternatively, push the Suspend button on the display's toolbar.

        STOPPED

        The command source is in a state where all telecommands are set back to an on-stack status. Adding new telecommands to the stack or removing some, however, is possible but dispatching is not. The command source may be set to this state manually or automatically after a telecommand failed to be released and the auto rejection mode is not enabled.
        Select the Stop item on the application button's menu when the display is activated. Alternatively, push the Stop button on the display's toolbar.

        In addition, this command source can operate in two different commanding modes:

        Commanding Mode

        Description

        DEFAULT

        This is the normal commanding mode allowing the dispatch of telecommands from multiple command sources.

        MASTER MANUAL

        This commanding mode prevents all other command sources from dispatching any telecommands.

        Note:
        The current commanding state and mode are displayed within the status bar of this command source.

    Telecommands & Telecommand Sequences

    There is a rich set of functions available in order to load, manipulate and dispatch telecommands or telecommand sequences. This also includes the possibility to create multiple templates for each telecommand as well as all the copy & paste operations which greatly simplify their handling on the stack.

    • Loading onto the Stack

      1. Open the TC Manual Stack Load Telecommand Dialog Box if it is not open yet:
        Select the Load... item on the application button's menu when the display is activated. Alternatively, push the Load button on the display's toolbar.
      2. Check either the TC Functions radio button to load a single telecommand or the TC Sequences radio button when a telecommand sequence is to be moved onto the stack.
      3. Decide whether the requested telecommand or telecommand sequence should be searched within the complete database or inside a predefined excerpt, called 'Group' by selecting the corresponding item in the Source combo box.
        Read more about how to work with groups:

        Creating a new Group

        Groups are a collection of telecommands that share some common properties like for example they share the same destination sub-system.

        1. Push the Groups... button. This opens the TC Manual Stack Telecommand Groups Dialog Box.
        2. Type a name for the telecommand group to be created into the Name combo box.
        3. Inside the Contents group box select the telecommand to get a member of this group inside the TC Functions combo box.
        4. Push the < button in order to add it to the group.
          Repeat the previous three steps in order to add more telecommands to a group.
        5. Push the OK button.

        Modifying a Group

        1. Push the Groups... button. This opens the TC Manual Stack Telecommand Groups Dialog Box.
        2. Select the name of the group to be modified inside the Name combo box. All telecommands belonging to the chosen groups are displayed within the Members list box.
        3. Perform the required changes:
          • Select a telecommand to be added inside the TC Functions combo box and push the < button.
          • Choose the member to deleted inside the Members group and push the > button. The >> button clears the complete group.
        4. Push the OK button.

        Deleting a Group

        1. Push the Groups... button. This opens the TC Manual Stack Telecommand Groups Dialog Box.
        2. Select the group to be deleted inside the Name combo box.
        3. Delete it by pushing the Remove button. The Remove All button clears all groups at once.
        4. Push the OK button.
        Note:
        Groups are not supported for telecommand sequences.
      4. If desired, select a filter to apply to the listed telecommands or telecommand sequences in order to reduce the number of available items to choose from.
        Read more about how to work with filters:

        Creating a new Search Filter

        Filters can be used to speed-up the search for telecommands or telecommand sequences.

        1. Push the Add... button. The TC Manual Stack Telecommand Filter Dialog Box will appear.
        2. Type a name for the new filter into the Name edit field.
        3. Specify a search key inside the Key combo box.
        4. Enter a corresponding value to be searched for into the Value edit field.
          Note:
          Multiple values are allowed and any text has to be enclosed in double quotes. Use either '&' (for a logical 'AND') or '|' (for an OR). There is no precedence in the evaluation of an expression; it is just processed from the left to the right.
        5. Push the Add button.
          Repeat the previous steps (except for the first one) if more than one search key should be added.
        6. Push the OK button.

        Modifying a Search Filter

        1. Select the filter to be modified inside the Filter combo box.
        2. Push the Modify... button. The TC Manual Stack Telecommand Filter Dialog Box will appear.
        3. Select the search key to be added or one those value needs to be changed in the Key combo box.
        4. Enter or modify the corresponding value inside the Value edit field.
          Repeat the previous two steps in order to alter more than one search key.
        5. Push the OK button.

        Deleting a Search Filter

        1. Select the filter to be deleted inside the Filter combo box.
        2. Push the Remove button. Using the Remove All button clears all filters.
      5. Select the telecommand or telecommand sequence inside the list box below the Filter combo box or type its name into the Load edit field.
      6. Push the Properties... button to define any release constraints or parameter values for the telecommand or telecommand sequence.
        The TC Manual Stack Telecommand Properties Dialog Box or TC Manual Stack Telecommand Sequence Properties Dialog Box will pop-up:

        Specifying the Properties for a Telecommand or Telecommand Sequence

        In most of the cases it is necessary to configure the properties of a telecommand or telecommand sequence before it can be dispatched. This includes the specification of the release constraints, on-board execution mode & time, the initialization of parameters and many more things.
        If a telecommand sequence needs to be set-up choose the corresponding entry inside the Entry combo box before proceeding.

            Constraint

            Description

            Override Pre-Execution Verification (PEV)

            Forces any pre-execution verification checks to result in a positive ('Go') result.

            Override Execution Verification (CEV)

            Forces any execution verification checks to result in a successful ('Passed') result.

            Disable PTV

            The TC Manual Stack Telecommand Release Constraints Dialog Box will pop-up:

            Specifying additional Release Constraints

            There are additional options available with respect to how and when the release of a telecommand should happen like, for example, grouping, blocking, time windows and interlocks.

            • Grouping

              Grouping is a method to force the command handler to release all those telecommands without any other interleaving ones.

              • Mark the Grouped check box in order to group the current telecommand with the previous one or vice versa to unlink it.

              Note:
              It is obvious that this option is available only if the current telecommand is part of a (temporary) sequence.

              Blocking

              Blocking is a method to pack multiple telecommands into a single telecommand packet.

              1. Mark the Blocked check box and push the Macro... button. This opens the TC Manual Stack Telecommand Block Macro Dialog Box:

                Adding a new Block Macro Member

                1. Select the telecommand to be added to the macro inside the Members combo box.
                2. Activate the position in the Macro list box after which the selected telecommand should be inserted and push the < button.
                  Repeat the previous steps if more than one member should be added.
                  Note:
                  The required size is always recalculated after modifying a block macro and, in case the resulting block is too big, a warning is issued.

                Deleting a Block Macro Member

                1. Select the telecommand to be removed from the macro inside the Members combo box.
                2. Push the > button.
                  Repeat the previous steps if more than one member should be deleted or use >> in order to clear the complete contents of the block macro.
                  Note:
                  The head telecommand of a block macro can never be deleted.
              2. Push the OK button.

              Note:
              This option is available only if the current telecommand is allowed to be, a so called, 'block head'.

              Defining a Time Window

              • Choose one of the following time window options available inside the Time Window group box:

                Option

                Description

                Between Immediately

                Immediately releases a telecommand without any associated time window. The release is considered as failed if it is not possible immediately after dispatch.

                ... and n s interval

                Tries to release a telecommand between the time t it was dispatched and t+n seconds if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window of n seconds is ignored and the release is considered as failed if it is not possible at time t.

                ... until t2

                Tries to release a telecommand between the time t1 it was dispatched and t2 if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window (t2-t1) is ignored and the release is considered as failed if it is not possible at time t1.

                Between hh:mm:ss after previous

                Tries to release a telecommand hh:mm:ss after the release of a previous one (at time t) without any associated time window. The release is considered as failed if it is not possible at time t+hh:mm:ss.
                If there is no previous telecommand hh:mm:ss is the offset with respect to the dispatch time.

                ... and n s interval

                Tries to release a telecommand hh:mm:ss after the release of a previous one (at time t) and t+n seconds if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window of n seconds is ignored and the release is considered as failed if it is not possible at time at time t+hh:mm:ss.
                If there is no previous telecommand hh:mm:ss is the offset with respect to the dispatch time.

                ... until t2

                Tries to release a telecommand hh:mm:ss after the release of a previous one (at time t1) and t2 if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window (t2-t1-hh:mm:ss) is ignored and the release is considered as failed if it is not possible at time t1+hh:mm:ss.
                If there is no previous telecommand hh:mm:ss is the offset with respect to the dispatch time.

                Between hh:mm:ss after dispatch

                Tries to release a telecommand hh:mm:ss after the dispatch time t without any associated time window. The release is considered as failed if it is not possible at time t+hh:mm:ss.

                ... and n s interval

                Tries to release a telecommand hh:mm:ss after the dispatch time t and t+n seconds if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window of n seconds is ignored and the release is considered as failed if it is not possible at time t+hh:mm:ss.

                ... until t2

                Tries to release a telecommand hh:mm:ss after the dispatch time t1 and t2 if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window (t2-t1-hh:mm:ss) is ignored and the release is considered as failed if it is not possible at time t1+hh:mm:ss.

                Between t1

                Tries to release a telecommand at time t1 without any associated time window. The release is considered as failed if it is not possible at time t1.

                ... and n s interval

                Tries to release a telecommand between time t1 and t1+n seconds if the release wait mode is enabled for the command source. If the release wait mode is not enabled the time window of n seconds is ignored and the release is considered as failed if it is not possible at time t1.

                ... until t2

                Creating, modifying and deleting Interlocks

                Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

                1. Type the name of the new interlock into the Name edit field or select an existing one in the associated combo box.
                2. Select the scope of the interlock inside the Properties group box:

                  Scope

                  Description

                  Global (all command sources)

                  Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

                  Local (same command source only)

                  Blocks the local command source as long as the interlock is in a pending or failed state.
                  Note:
                  Enable the No wait state option if the interlock should not block when in a pending state.

                  Local on specified subsystem

                  Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
                  Note:
                  Enable the No wait state option if the interlock should not block when in a pending state.

                  Stage

                  Description

                  Reception (at ground station)

                  The interlock is set depending on the outcome of the transmission of a telecommand. It is set to 'Success' if a telecommand was successfully uplinked or to 'Failure' otherwise.

                  Acceptance

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing an 'Acceptance' stage. Otherwise it remains in an idle state.

                  Start

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Start' stage. Otherwise it remains in an idle state.

                  Progress

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Progress' stage. Otherwise it remains in an idle state.

                  Completion

                  The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Completion' stage. Otherwise it remains in an idle state.

      Any telecommand or telecommand sequence that needs to be released must be dispatched from the command source through a two-step ARM & GO procedure.

      • Choose the telecommand(s) or telecommand sequence that should be dispatched by clicking onto it (them) and then select:
        • The ARM item on the application button's menu. Alternatively, push the ARM button on the display's toolbar.
          Arms the selected telecommand(s) or telecommand sequence on the stack.
          Arming already armed telecommands again will un-arm them.
          Note:
          The background color will change to indicate the armed status of the telecommand(s) or telecommand sequence.
        • The GO item on the application button's menu. Alternatively, push the GO button on the display's toolbar.
          Dispatches the selected telecommand(s) or telecommand sequence on the stack. This must happen within the dispatch time window specified for this command source otherwise the telecommands will be un-armed automatically.
          Note:
          Again, the background color will change to indicate the dispatched status.

      Note:
      The manual dispatch method can only be omitted when enabling the Pre-Authorized option belonging to the release constraints for a telecommand.

      Undispatching

      Usually, there is no need to undispatch telecommand(s) or telecommand sequences. Nevertheless it is possible to do so under certain circumstances:

      1. Change the commanding state of the command source to 'SUSPENDED'.
      2. Choose those telecommand(s) or telecommand sequence to be undispatched and select the Undispatch item on the application button's menu. Alternatively, push the Undispatch button on the display's toolbar.
        This will remove the dispatched telecommands from the release queue of the command handler and change their status to on-stack.

      Note:
      Switching a command source to a 'STOPPED' commanding state reverts all dispatched telecommands back to an on-stack status.

      Acknowledging

      There are many reasons that might require a telecommand to be acknowledged: A failure during release, confirmation of a stage verification check result and others.
      All telecommands requiring a certain attention are marked with a small icon inside the No. column of the command source. A description of the reason can be viewed through the tool-tip attached to that icon.

      • Choose the telecommand to be acknowledged by clicking onto it and selecting the Acknowledge item on the application button's menu when the display is activated. Alternatively, push the Acknowledge button on the display's toolbar.

      Note:
      Sometimes, it might be necessary to acknowledge a telecommand multiple times. This is because the command source maintains a 'Last-In-First-Out' (LIFO) stack for each telecommand, able to keep an unlimited amount of messages for viewing.

      Searching

      Each stack-based command source provides help when searching for a telecommand. Thereby, every column can serve as a search key.

      1. Open the TC Manual Stack Search Telecommand Dialog Box if it is not open yet:
        Click onto the column header representing the requested search key.
      2. Enter the searched value into the Find edit field or select it inside the combo box (for predefined or recent values).
        Note:
        Some search keys require different ways to specify a searched value.
      3. Determine the direction the search should take place by checking either the Up or Down radio button.
        Note:
        The search always starts with the first or last telecommand available depending on the search direction if the Search all check box is marked.
      4. Choose one or more of the available search options:

        Option

        Description

        Search all

        Searches all telecommands listed. Starts the search either from the first or last parameter (depending on the search direction) but not from the current one.

        Match case

        Searches for a word or phrase with matching case.

        Match whole word

        Searches for the specified word which may not be part of another word.
      5. Push the Find Next button to start or continue the search.
        The current telecommand complying with the searched value is always marked with a '*' within the 'No.' column.
      6. Push the Close button to terminate the search.

    Telecommand Stacks

    Telecommand stacks are nothing else than predefined portions of the stack saved to a repository for later re-use.

    • Saving into a Repository

      1. Select all those telecommands that should become part of the saved stack.
      2. Open the TC Manual Stack Save Telecommand Stacks Dialog Box if it is not open yet:
        Select the Save... item on the application button's menu when the display is activated. Alternatively, push the Save button on the display's toolbar.
      3. Enter a name for the selected portion of the stack into the Name edit field.
      4. Type a short description into the Description edit field summarizing the purpose of this saved stack. This simplifies future searches.
      5. Choose the appropriate storing options:

        Option

        Description

        Use relative release times

        Converts absolute telecommand release times into relative ones i.e. the release time of all telecommands will be converted to a delta time with respect to the first one. This greatly simplifies a later re-use.

        Use relative on-board execution times

        Converts absolute on-board execution times into relative ones with the delta execution time relative to the execution time (or release time if no execution time is available) of the first telecommand.
      6. Mark the Validated check box if no consistency check with respect to the database should be performed when this stack is loaded at a later time.
        Note:
        Leave this option disabled in order to make sure that all telecommands are checked against the database on the workststion loading the saved stack.
      7. Inside the Repository group box enter a file name for the repository to store the stack into the Filename edit field.
        Note:
        The specified file may exist already and contain other saved stacks. Push the Inspect button in order to examine its current contents.
      8. Push the OK button.

      Loading from a Repository

      1. Open the TC Manual Stack Load Telecommand Stacks Dialog Box if it is not open yet:
        Select the Open... item on the application button's menu when the display is activated. Alternatively, push the Open button on the display's toolbar.
      2. Enter the file name of the repository into the Filename edit field.
        Use the Browse... button to search for an existing file.
      3. Push the Inspect button to display its contents inside the upper list box inside the Properties group box.
        If the repository contains many saved stacks there is a way to use filters:

        Creating a new Filter

        Filters can be created to speed-up the search for saved stacks within repository files.

        1. Push the Add... button. The TC Manual Stack Telecommand Filter Dialog Box will appear.
        2. Type a name for the new filter into the Name edit field.
        3. Specify a search key inside the Key combo box.
        4. Enter a corresponding value to be searched for into the Value edit field.
          Note:
          Multiple values are allowed and any text has to be enclosed in double quotes. Use either '&' (for a logical 'AND') or '|' (for an OR). There is no precedence in the evaluation of an expression; it is just processed from the left to the right.
        5. Push the Add button.
          Repeat the previous steps (except for the first one) if more than one search key should be added.
        6. Push the OK button.

        Modifying a Filter

        1. Select the filter to be modified inside the Filter combo box.
        2. Push the Modify... button. The TC Manual Stack Telecommand Filter Dialog Box will appear.
        3. Select the search key to be added or one those value needs to be changed in the Key combo box.
        4. Enter or modify the corresponding value inside the Value edit field.
          Repeat the previous two steps in order to alter more than one search key.
        5. Push the OK button.

        Deleting a Filter

        1. Select the filter to be deleted inside the Filter combo box.
        2. Push the Remove button. Using the Remove All button clears all filters.

        Applying Filters

        Filters can be used to speed-up the search for saved stacks within repository files.

        • Select the filter to be used in order to search the contents of the repository inside the Filter combo box.
          This will automatically update the saved stack list according to the filter criteria.
        Note:
        Each saved stack can be viewed in more detail by selecting it in the upper list box. Its contents can then be browsed in the lower Contents list box.
      4. Select the saved stack to be loaded inside the upper list box of the Properties group box and eventually push the Validate button in order to make sure that the saved stack is valid with respect to the current database. If it is not the case and the saved stack should nevertheless be loaded then mark the Load anyway check box.
      5. Push the OK button.

      Removing from a Repository

      1. Open the TC Manual Stack Save Telecommand Stacks Dialog Box if it is not open yet:
        Select the Save... item on the application button's menu when the display is activated. Alternatively, push the Save button on the display's toolbar.
      2. Inside the Repository group box enter a file name for the repository into the Filename edit field.
        Use the Browse... button to search for an existing file.
      3. Push the Inspect button. This lists the contents of the repository in the lower list box.
      4. Select the saved stack to be removed from the repository and then push the > button or use the >> button to empty it.
      5. Push the OK button.

      Deleting a Repository

      1. Open the TC Manual Stack Load Telecommand Stacks Dialog Box if it is not open yet:
        Select the Open... item on the application button's menu when the display is activated. Alternatively, push the Open button on the display's toolbar.
      2. Enter the name of the repository file into the Filename edit field.
        Use the Browse... button to search for an existing file.
      3. Push the Delete button.
      4. Push the OK button.

    Interlocks

    Interlocks are essential when it comes to coordinate the release of telecommands from a command source. They can allow or prevent the release of telecommands depending on the success/failure of previous telecommands. Global interlocks apply to all command sources whereas local ones affect only those command sources hosting them.

    • Creating, modifying and deleting Interlocks

      Interlocks can be manipulated when telecommands are loaded onto the stack or when they are modified.

      1. Load or modify the telecommand to interlock or to be interlocked by:
        • Selecting the Load... item on the application button's menu when the display is activated or, alternatively, push the Load button on the display's toolbar. Then choose the desired telecommand in the dialog box and push the Properties... button.
        • Activating the telecommand on the stack and selecting the Edit è Properties... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
        This opens the TC Manual Stack Telecommand Properties Dialog Box.
      2. Push the Release... button. The TC Manual Stack Telecommand Release Constraints Dialog Box will appear.
      3. Push the Interlocks... button in order to create, modify or delete interlocks with the TC Manual Stack Telecommand Release Interlocks Dialog Box.
      4. Type the name of the new interlock into the Name edit field or select an existing one in the associated combo box.
      5. Select the scope of the interlock inside the Properties group box:

        Scope

        Description

        Global (all command sources)

        Blocks all command sources and prevents the release of telecommands as long as the interlock is in a pending or failed state.

        Local (same command source only)

        Blocks the local command source as long as the interlock is in a pending or failed state.
        Note:
        Enable the No wait state option if the interlock should not block when in a pending state.

        Local on specified subsystem

        Blocks the release of all telecommands belonging to the specified sub-system of the local command source as long as the interlock is in a pending or failed state.
        Note:
        Enable the No wait state option if the interlock should not block when in a pending state.

        Local on specified telecommands

        Blocks the release of all telecommands associated with this interlock of the local command source as long as the interlocking telecommand has not been verified successfully.
      6. Choose the stage the interlock applies to:

        Stage

        Description

        Reception (at ground station)

        The interlock is set to Success if the telecommand was successfully received by the ground station command uplink infrastructure or to 'Failure' otherwise.

        Transmission

        The interlock is set depending on the outcome of the transmission of a telecommand. It is set to 'Success' if a telecommand was successfully uplinked or to 'Failure' otherwise.

        Acceptance

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing an 'Acceptance' stage. Otherwise it remains in an idle state.

        Start

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Start' stage. Otherwise it remains in an idle state.

        Progress

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Progress' stage. Otherwise it remains in an idle state.

        Completion

        The interlock is set to either 'Success' or 'Failure' depending on the outcome of a telecommand execution verification check containing a 'Completion' stage. Otherwise it remains in an idle state.
      7. Push the Add button in order to add a new interlock or Remove to delete an existing one. Pushing Remove All clears all at once.
        Notes:
        • Global interlocks can only be created, modified or deleted when the privilege 'Performing privileged tasks on the manual telecommand stack' is enabled.
        • Only interlocks in an idle, successful or failed state can be modified or deleted.
      8. Push all necessary OK buttons.

      Enabling or disabling Interlocks

      • Select the Interlock Checks item on the application button's menu when the display is activated. Alternatively, push the Interlock Checks on the display's toolbar.
        This enables or disables the use of local interlocks.

      Note:
      Disabled interlocks have no effect.

      Resetting Interlocks

      1. Select the Reset Interlocks... item on the application button's menu when the display is activated. Alternatively, push the Reset Interlocks on the display's toolbar.
        This will open the TC Manual Stack Reset Interlocks Dialog Box.
      2. Choose the interlock to be reset to idle state by selecting it in the list and by pushing the Reset button. Use the Reset All button to reset all of them at once.
        Note:
        Interlocks in a pending state can not be reset manually.
      3. Push the OK button.

      Note:
      This option is not available if there are no interlocks to be reset.

Print Data

  1. Activate the TC Manual Stack to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print the TC Manual Stack. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever the TC Manual Stack needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

Close it

  • Push the Manual TC Stack toggle-button inside the Command Sources panel of the Telecommand ribbon bar category or select the Close item on the display's system menu.

Note:
The TC Manual Stack will continue to schedule and release telecommands even when it is in a closed (hidden) state.

Prerequisites:
The privilege 'Changing the contents of the manual telecommand stack' must be enabled to manipulate the TC Manual Stack and to release telecommands from this command source. Having only the 'Browsing the contents of the manual telecommand stack' privilege active prevents any changes. Some critical actions and configuration changes require the 'Performing privileged tasks on the manual telecommand stack' privilege.
