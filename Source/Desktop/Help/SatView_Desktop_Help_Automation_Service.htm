Automation Service

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ exposes various interfaces via its Automation Service. This allows registered users to enable/disable, execute, terminate services on a workstation or change the settings of a service from a remote site.

How do I...

Configure the Automation Service

  1. Open the Automation Service Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Automation panel of the System ribbon bar category.
  2. Specify the TCP/IP port number and memory buffer size to be allocated for the service by adjusting the numbers in the Port and Buffer spin boxes. A time interval affecting connection and sending timeouts has to be specified in the Timeout spin box.
  3. Mark the Disabled button if the service should be deactivated.

  4. Adding a Client

    1. Enter the name of the new client into the Username edit field.
      Note:
      A client does not have to be human; it can also be another piece of software.
    2. Type a password into the Password edit field and re-type it into the Confirm field.
    3. Mark the Encrypt check box if the password should be encrypted during communications.
    4. Choose the permission to be granted to the client by selecting it inside the Permissions combo box and by pushing the Allow button afterwards.
      Repeat the previous step if multiple permissions should be assigned to the client.
      Use the Revoke button to remove a selected permission or push the Revoke All button to delete all permissions.
    5. Push the Add button to add the new client to the list of authorized users of the service.
      Note:
      Use the Ban Client check box to prevent a client from accessing the automation service.

    Removing a Client

    1. Select the client to be removed from the list of authorized users inside the corresponding list view.
    2. Push the Remove button or use the Remove All button to clear the complete list of authorized users of this service.
  5. Push the OK button.

Prerequisites:
The privilege 'Changing the automation service settings' must be enabled to modify the configuration of the automation service.
