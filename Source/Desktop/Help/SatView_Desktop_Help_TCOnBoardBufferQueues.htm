TC On-Board Buffer Queues Display

SUMMARY

Saturday, October 10, 2015
Version 1.0

The TC On-Board Buffer Queues Display reflects the status of the on-board telecommand buffer queues. It offers an overview over all on-board queues as well as a detailed representation of their contents. In addition, assistance for the generation of telecommands necessary to control those buffer queues is granted too.

How do I...

Open the TC On-Board Buffer Queues Display

  1. Push the Buffer Queues toggle-button inside the On-Board Services panel of the Telecommand ribbon bar category or, alternatively, open the Windows button's menu inside the Window panel of the System ribbon bar category and select the TC On-Board Buffer Queues item.
  2. Optionally configure the TC On-Board Buffer Queues display.

  • Understand the Layout

Configure the Settings

Many aspects of the TC On-Board Buffer Queues display are configurable. Layout and the print profile can be modified as required.

  1. Open the TC On-Board Buffer Queues Settings Dialog Box if it is not open yet:
    Activate the TC On-Board Buffer Queues display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.

  2. Read more about:

    Modifying the Layout

    Specifying the Appearance:

    1. Select the Layout page.
    2. Choose the pane which the changes should apply to in the For Pane combo box.

    3. Creating a new Column

      1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
      2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
      3. Push the Insert After button.
        Note:
        The No. or Name column respectively must always be the left most one and it cannot be deleted.

      Deleting a Column

      1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
      2. Push the Remove button.
        Note:
        The No. or Name column respectively cannot be deleted.

      Specifying the Background Grid & Color

      • Mark the Grid check box in order to enable a background grid and select the desired color inside the associated color combo box.

      Changing the Fonts

      1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the chosen pane.
      2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.

    Controlling the Operating Flags:

    1. Select the Layout page.

    2. Tool-tips

      A small tool-tip containing on-board buffer queue or telecommand related details can be displayed on request. When this option is enabled and the cursor is moved over a part containing additional information, a small hand occurs indicating that a tool-tip is available. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

      • Mark the Tooltips check box in order to enable this feature.

    Modifying the Print Profile

    1. Select the Printing page.
    2. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
    3. Inside the Options group box choose which on-board buffer type should be printed by selecting it in the Buffer combo box.
    4. Mark the Summary check box if a list of all buffer queues of the selected type has to be included in the print-out.
    5. Choose the Queue check box to print the contents of a specific on-board buffer queue. The queue must be selected in the associated combo box. When this option is selected, it is possible to restrict the amount of data to be printed by choosing the appropriate option inside the Contents group box:

      Range

      Description

      No. from n1 to n2

      Prints all on-board buffer queue entries between number n1 and n2.

      Execution Time from t1 to t2

      Prints all on-board buffer queue entries with an execution time between t1 and t2.

      Note:
      If several filter options are selected they are combined through a logical 'AND'.

    6. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
  3. Push the OK button.

Explore Features

    On-Board Buffer Queues:

    Controlling a Queue

    The TC On-Board Buffer Queues display can help to generate and customize the telecommands necessary to manage the on-board buffer queues.

    • Starting/Resuming

      This dialog box allows to prepare the necessary telecommand to start or resume an on-board buffer queue.

      1. Open the TC On-Board Buffer Queues Start Properties Dialog Box if it is not open yet:
        Select the Start... item on the application button's menu when the display is activated. Alternatively, push the Start button on the display's toolbar.
      2. Specify the on-board execution mode by checking either the Immediately or At Time radio button. In the latter case the future on-board execution time has to be entered into the associated time spin-box.
      3. Choose the command source from which the prepared telecommand should be dispatched from by checking either the Manual Stack or Automatic Stack radio button.
      4. Decide if the telecommand has to be released immediately or at a specific time t by selecting the Immediately or At Time radio button and eventually entering the time t.
      5. Mark the Pre-Authorized check box in order to force an automatic dispatch from the chosen command source.
      6. Push the OK button.

      Note:
      This feature is not available if the spacecraft does not support the operation.

      Dumping

      This dialog box allows to prepare the necessary telecommand to dump the contents of an on-board buffer queue.

      1. Open the TC On-Board Buffer Queues Dump Dialog Box if it is not open yet:
        Select the Dump... item on the application button's menu when the display is activated. Alternatively, push the Dump button on the display's toolbar.
      2. Choose what to dump from the selected on-board buffer queue by checking either of these options:

        Option

        Description

        All entries

        Downloads the complete contents of the selected on-board buffer queue.

        From entry no.: n1 to n2

        Requests to dump a range spanning from entry n1 to n2 only.

        Entries between t1 and t2

        Requests to dump only those buffered telecommands with an on-board execution time between t1 and t2.

        All entries belonging to the sub-system x

        Dumps all buffered telecommands related to the specified sub-system x only.

        The entry with sequence number n

        This option can be combined with one of the three previous ones and restricts the dump to the telecommand with a specific sequence number n.

        Note:
        Some of the above options may not be available i.e. the spacecraft does not support that operation.

      3. Mark the Synchronize the on-ground buffer queue check box if the contents of the selected on-board buffer queue should be updated automatically according to the dumped data.
        Note:
        All downloaded telecommands are marked with a special dump status symbol indicating that they are confirmed to be buffered on-board. Through this indication it is possible to manually delete all those telecommands that are not confirmed. Special care however has to be given to the situation where only a sub-set of the on-board buffer queue is requested to be dumped and hence not all telecommands will be tagged although they exist on-board.
      4. Select the Clear the dump status of all entries option in order to clear the status from previous dumps. This gives a better overview about which telecommands are going to be dumped by the current request.
      5. Push the Send... button in order to prepare the telecommand for the requested dump. This opens the TC On-Board Buffer Queues Dump Properties Dialog Box:

        Finalizing the Dump Request

        1. Specify the on-board execution mode by checking either the Immediately or At Time radio button. In the latter case the future on-board execution time has to be entered into the associated time spin-box.
        2. Choose the command source from which the prepared telecommand should be dispatched from by checking either the Manual Stack or Automatic Stack radio button.
        3. Decide if the telecommand has to be released immediately or at a specific time t by selecting the Immediately or At Time radio button and eventually entering the time t.
        4. Mark the Pre-Authorized check box in order to force an automatic dispatch from the chosen command source.
        5. Push the OK button.
      6. Push the OK button.

      Note:
      This feature is not available if the spacecraft does not support the operation.

    Acknowledging a Queue

    There are several reasons why an on-board buffer queue may be in an alerted state like, for example, a discrepancy between the real-time telemetry data and the current contents of the on-board buffer queue represented on-ground.

    • Activate the alerted on-board buffer queue in the upper pane and select the Acknowledge item on the application button's menu when the display is activated. Alternatively, push the Acknowledge button on the display's toolbar.

    Note:
    Sometimes, it might be necessary to acknowledge an on-board buffer queue multiple times. This is because the on-board buffer queue list maintains a 'Last-In-First-Out' (LIFO) stack for each queue, able to keep an unlimited amount of messages for viewing.

    Queued Telecommands:

    Browsing the Properties

    1. Open the TC On-Board Buffer Queues Telecommand Properties Dialog Box or TC On-Board Buffer Queues Telecommand Sequence Properties Dialog Box if it is not open yet:
      Activate the TC On-Board Buffer Queues display, choose the on-board buffer queue containing the telecommand (sequence) to be browsed in the upper pane, select it in the lower pane and choose the Details... item on the application button's menu. Alternatively, push the Details button on the display's toolbar.
      When a telecommand sequence is browsed, select the required member inside the Entry combo box.
    2. Check for the requested telecommand properties.
      Notes:
      • Double-click onto one of the parameters in the list box inside the Parameters group box to view its associated calibration table (if any).
      • Examine the data inside the Raw Data group box when looking for the encoded telecommand data.
    3. Push the OK button.

    Acknowledging

    There are many reasons that might require a telecommand to be acknowledged: A failure during on-board execution, confirmation of a stage verification check result and others.
    All telecommands requiring a certain attention are marked with a small icon inside the No. column of the on-board buffer queue contents. A description of the reason can be viewed through the tool-tip attached to that icon.

    • Activate the on-board buffer queue containing the alerted telecommand in the upper pane, click onto the telecommand inside the lower pane and select the Acknowledge item on the application button's menu. Alternatively, push the Acknowledge button on the display's toolbar.

    Note:
    Sometimes, it might be necessary to acknowledge a telecommand multiple times. This is because the on-board buffer queue contents maintains a 'Last-In-First-Out' (LIFO) stack for each telecommand, able to keep an unlimited amount of messages for viewing.

    Searching

    The on-board buffer queue contents provides help when searching for a telecommand. Thereby, every column can serve as a search key.

    1. Open the TC On-Board Buffer Queues Search Telecommand Dialog Box if it is not open yet:
      Click onto the column header representing the requested search key.
    2. Enter the searched value into the Find edit field or select it inside the combo box (for predefined or recent values).
      Note:
      Some search keys require different ways to specify a searched value.
    3. Determine the direction the search should take place by checking either the Up or Down radio button.
      Note:
      The search always starts with the first or last telecommand available depending on the search direction if the Search all check box is marked.
    4. Choose one or more of the available search options:

      Option

      Description

      Search all

      Searches all telecommands listed. Starts the search either from the first or last parameter (depending on the search direction) but not from the current one.

      Match case

      Searches for a word or phrase with matching case.

      Match whole word

      Searches for the specified word which may not be part of another word.
    5. Push the Find Next button to start or continue the search.
      The current telecommand complying with the searched value is always marked with a '*' within the 'No.' column.
    6. Push the Close button to terminate the search.

    Specifying the Display Options

    There are multiple ways how to control the amount of details displayed for a telecommand scheduled for on-board execution:

    Changing the columns of the on-board buffer queue contents:

    1. Open the TC On-Board Buffer Queues Settings Dialog Box if it is not open yet:
      Activate the TC On-Board Buffer Queues display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
    2. Select the Layout page.
    3. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
    4. Click at the position on the column items bar below the No. combo box where the selected column should be inserted. A '*' indicating the selected column appears.
    5. Push the Insert After button in order to add a column or Remove to delete a selected column.
      Note:
      The No. column must always be the left most one and it cannot be deleted.
    6. Push the OK button.

    Controlling the number of arguments:

    • Select the Full View Mode item on the application button's menu when the display is activated or, alternatively, push the Full View Mode button on the display's toolbar.

    By enabling this option a telecommand is split into as many lines on the stack as are required to display all arguments involved.

Print Data

  1. Activate the TC On-Board Buffer Queues display to be printed.
  2. Choose one of the following printing methods:

    Method

    Description

    How to do it

    Quick Printing

    This method is the most simple one because it does not need any further interaction of the user in order to print the TC On-Board Buffer Queues display. The way how the display is printed is defined by the Default print job profile.

    Push the Print button on the display's toolbar.

    Interactive Printing

    Whenever the TC On-Board Buffer Queues display needs to be printed in a different way than defined by the Default print job profile this method is the one to be used. It allows the modification of the print properties without changing the profile.
    • Select the Print... item on the application button's menu when the display is activated.
    • Change the settings in the Print Dialog Box as required.
    • Push the OK button.

Close it

  • Push the Buffer Queues toggle-button inside the On-Board Services panel of the Telecommand ribbon bar category or select the Close item on the display's system menu.

Note:
The TC On-Board Buffer Queues display will continue to update the representation of the on-board buffer queues even when it is in a closed (hidden) state.

Prerequisites:
The privilege 'Changing the telecommand on-board software services' must be enabled to configure the TC On-Board Buffer Queues display. Having only the 'Browsing the telecommand on-board software services' privilege active prevents any changes.
