PFLP Service

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ supports a special protocol called Packet Front-End Link Protocol (PFLP) designed to export real-time telemetry data in form of telemetry source packets. Each SatView™ workstation can handle an unlimited amount of clients and provides an easy way how to control the availability of the service including client access privileges.

How do I...

Configure the PFLP Service

  1. Open the PFLP Service Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Packet Front-End Link Protocol panel of the Communication ribbon bar category.
  2. Choose the device to be used for the service inside the Use Device combo box.
  3. Eventually configure the selected device by pushing the Configure... button.
    Note:
    The configuration is effective on a per-device basis and stored as part of the PFLP Service data.
    The available configuration options depend on the device type selected. Consult the chapters related to the chosen device for more details.
  4. Choose the appropriate option in the Availability group box:

    Option

    Description

    Unlimited

    The PFLP Service is available all the time.

    Daily between t1 and t2

    Restricts the availability of the PFLP Service to a daily period between t1 and t2.

    Disabled

    Disables the PFLP Service.

    Working Days only

    The PFLP Service is available in the specified way between Monday and Friday only.
    Note:
    This option can be combined with one of the above.
    Note:
    Mark the Audit check box if an event message should be issued whenever the availability of the service changes.
  5. Push the OK button.

Prerequisites:
The privilege 'Changing the PFLP service settings' must be enabled in order to modify the settings of the service.

Browse PFLP Connections

Each SatView™ workstation can act as a PFLP server or client. It is possible to see how many clients are currently requesting telemetry data via a PFLP connection or to terminate one or more of those sessions.

  1. Open the PFLP Clients Dialog Box if it is not open yet:
    Push the Clients button inside the Packet Front-End Link Protocol panel of the Communication ribbon bar category.
  2. Select the PFLP connection those details should be displayed in the Clients list box.
  3. Push the OK button.

Prerequisites:
The privilege 'Viewing the existing PFLP connections' must be enabled in order to view PFLP connection details.

Close PFLP Connections

Each SatView™ workstation can act as a PFLP server or client. It is possible to see how many clients are currently requesting telemetry data via a PFLP connection or to terminate one or more of those sessions.

  1. Open the PFLP Clients Dialog Box if it is not open yet:
    Push the Clients button inside the Packet Front-End Link Protocol panel of the Communication ribbon bar category.
  2. Select the PFLP connection to be closed in the Clients list box. Push the Close button.
    Use the Close All button to close all existing PFLP connections.
  3. Push the OK button.

Prerequisites:
The privilege 'Closing the existing PFLP connections' must be enabled in order to close PFLP connections.
