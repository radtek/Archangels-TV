TC History Settings Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

Many aspects of the TC History Display are configurable. Layout, filtering & logging options and the print profile can be modified as required.

How do I...

Modify the Layout

  1. Open the TC History Settings Dialog Box if it is not open yet:
    Activate the TC History display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Layout page.
  3. Specifying the Appearance:

    Creating a new Column

    1. Inside the Format group box choose the column to be inserted by selecting the corresponding name in the Name combo box.
    2. Click at the position on the column items bar below the Name combo box where the selected column should be inserted. A '*' indicating the selected column appears.
    3. Push the Insert Before or Insert After button.
      Note:
      The No. column must always be the left most one and it cannot be deleted.

    Deleting a Column

    1. Select the column to be deleted by clicking on it in the column items bar below the Name combo box. A '*' indicating the selected column appears.
    2. Push the Remove button.
      Note:
      The No. column cannot be deleted.

    Specifying the Background Grid & Color

    • Mark the Grid check box in order to enable a background grid and select the desired color inside the associated color combo box.

    Changing the Fonts

    1. Inside the Fonts group box select the Title or Items radio button depending on which font should be changed for the area where the historical telecommands are displayed.
    2. Select the font name in the Name combo box and select the desired style and size in the Style combo box and Size edit field respectively. A sample of the chosen font is always displayed in the Sample area.
    Controlling the Operating Flags:

    Tool-tips

    A small tool-tip containing telecommand related details can be displayed on request. When this option is enabled and the cursor is moved over a telecommand inside the corresponding area, a small hand occurs indicating that a tool-tip is available for that part of the telecommand. Press the right mouse button to make it pop-up. The tool-tip disappears automatically when the mouse button is released.

    • Mark the Tooltips check box in order to enable this feature.
  4. Push the OK button.

Change the Filtering Options

Filters with respect to various criteria can be set in place in order to restrict the telecommands displayed.

  1. Open the TC History Settings Dialog Box if it is not open yet:
    Activate the TC History display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Filtering page.
  3. Choose the desired filtering criteria:

    Filter Option

    Description

    Telecommand

    Applies a filter with respect to properties of telecommands:

    Property

    Description

    Name

    Filters all telecommands with a specified name i.e. all telecommands with a different name are not displayed.

    Sequence

    Filters all telecommand sequences with a specified name.

    Type

    Filters all telecommands with either an Immediate, Time-Tagged, 1st Step or 2nd Step type.

    Sub-system

    Filters all telecommands related to a certain sub-system:

    1. Select the desired sub-system in the Sub-system combo box.
    2. Push the < button. This adds it to the list of sub-systems.
      Repeat the previous steps to add more or use the > button to remove a selected entry.

    Verification Status

    Filters all telecommands those stage verification check ended in a certain result. The following stage verification checks can be compared with these predefined results:

    Stage

    Result

    Transmission
    Acceptance
    Start
    Progress
    Completion

    Passed
    Failed
    Unknown
    Unverified
    1. Select the stage verification check in the Stage combo box.
    2. Mark the State check box and choose the required check result in the associated combo box.
      Repeat the previous steps if more that one stage check should be included in the filter.

    Note:
    Telecommands not implementing a stage check that is included in the filter are not listed.

    Limited to maximal n entries

    Limits the number of telecommands listed to n entries.

    Note:
    Choosing multiple filtering options results in combining them with a logical 'AND'.

  4. Choose the desired sorting order:

    Sorting Option

    Description

    By Transmission Time

    The telecommands are listed according to their transmission time.

    By Execution Time

    The telecommands are listed according to their on-board execution time. If there is no on-board execution time attached to the telecommand the transmission time is used instead.

    Show the latest entry as first one

    Determines if the latest telecommands are inserted at the top or appended at the bottom.
  5. If required, add an alert behavior concerning the telecommand stage verification checks or for requested acknowledgments by pushing the Alerts... button.
    This opens the TC History Alert Options Dialog Box:

    Specifying the Alert Behavior

    1. Select the type of incident that should be alerted inside the Type combo box:

      Incident

      Description

      Any telecommand verification failure

      Generates an alert whenever a stage verification check failed for a telecommand.

      Telecommand acceptance failure

      An alert is raised when the acceptance stage verification check failed for a telecommand.

      Telecommand start failure

      An alert is raised when the start stage verification check failed for a telecommand.

      Telecommand progress failure

      An alert is raised when the progress stage verification check failed for a telecommand.

      Telecommand completion failure

      An alert is raised when the completion stage verification check failed for a telecommand.

      Acknowledgeable telecommand acceptance success

      An alert is raised when a telecommand requires the acknowledgment for its successful acceptance on-board.

      Acknowledgeable telecommand start success

      An alert is raised when a telecommand requires the acknowledgment for its successful execution start on-board.

      Acknowledgeable telecommand progress success

      An alert is raised when a telecommand requires the acknowledgment for its successful execution progress on-board.

      Acknowledgeable telecommand completion success

      An alert is raised when a telecommand requires the acknowledgment for its successful execution completion on-board.
    2. Inside the Action group box select the desired audition through the Audition combo box that should be performed when the above selected incident occurs. It will turn up as an event in the Global Eventbox with the color specified in the associated Color combo box. Mark the Blinking & audible check box if the event should also blink (incl. audible sound if chosen) and require an acknowledgment.
    3. Add all notifications to be performed (if any) when an incident of the selected type occurs by choosing them first in the Notifications combo box and then by pressing the Add button.
      Use the Remove button to delete the selected notification profile or Remove All to delete all of them at once.
    4. Push the OK button.
  6. Push the OK button.

Specify the Retrieval Source

  1. Open the TC History Settings Dialog Box if it is not open yet:
    Activate the TC History display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Retrieving page.
  3. Choose if either the transmission time or the on-board execution time should be used as reference time by checking the Transmission Time or Execution Time check box.
    Note:
    If the on-board execution time is selected as reference time for retrievals and an archived telecommand does not have an on-board execution time (immediate on-board execution) then the transmission time is used instead.
  4. Select the retrieval file to become the source for any retrieval requests by highlighting it in the Data Source combo box.
    Push the Inspect button to check a potential retrieval file. The details about the inspected retrieval file are shown in the Source Information group box.
    Note:
    The set of retrieval files listed in the Data Source combo box can be specified in the Retrievals page of the System Settings Dialog Box.
  5. Push the OK button.

Modify the Print Profile

  1. Open the TC History Settings Dialog Box if it is not open yet:
    Activate the TC History display and select the Settings... item on the application button's menu. Alternatively, push the Settings button on the display's toolbar.
  2. Select the Printing page.
  3. Select the print profile to be used inside the Print Profile combo box. The print profile specifies the printer and paper related properties.
  4. Inside the Options group box choose those telecommands that should be printed:

    Range

    Description

    No. from n1 to n2

    Prints all telecommands between number n1 and n2.

    Release Time from t1 to t2

    Prints all telecommands between time t1 and t2.

    Execution Time from t1 to t2

    Prints all telecommands with an on-board execution time between time t1 and t2.
    Note:
    If this option is selected and an archived telecommand does not have an on-board execution time (immediate on-board execution) then the transmission time is used instead.

    Overall Verification:
    Success
    Failure

    Prints all telecommands with a verification stage check result of either 'Success' or 'Failure'.
    Note:
    If a telecommand has no execution verification check attached, a 'Success' result is assumed.

    Note:
    If several filter options are selected they are combined through a logical 'AND'.

  5. Type a comment into the Text edit field if something should be added to the title of the printout. Mark the Clear after use check box if this text should be used on one printout only.
  6. Push the OK button.

Prerequisites:
The privilege 'Changing the configuration of telecommand history display windows' must be enabled.
