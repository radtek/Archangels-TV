Notifications

SUMMARY

Saturday, October 10, 2015
Version 1.0

Notification profiles are used to specify different notification behaviors. Each notification profile has a unique name and its use greatly simplifies the specification of who should be notified in case of an event and also how this should take place.
The Default notification profile describes the default notification behavior. It can be modified but not deleted.

How do I...

Create a new Notification Profile

  1. Open the System Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Configuration panel of the System ribbon bar category.
  2. Select the Notifications page.
  3. Enter the name of the new notification profile into the Name edit field.
    Note:
    The name of the notification profile must be unique.
  4. Choose how a notification should take place by selecting either the None, E-Mail, FAX, Phone Call, Pager Call, FTP Message or Printer Message radio button:

    Type

    Description

    None

    No notification is performed.
    Note:
    No further settings are required for this option.

    E-Mail

    Sends an e-mail to the specified users.
    Note:
    The contents of this mail is determined by the event causing the notification but a common prolog and epilog text can be specified when required.
    At least one recipient must be specified.

    FAX

    Sends a fax to the specified users.
    Note:
    The contents of this fax is determined by the event causing the notification but a common prolog and epilog text can be specified when required.
    At least one recipient must be specified.

    Phone Call

    Makes a phone call to the specified users.
    Note:
    The contents of this call is determined by the event causing the notification but a common prolog and epilog text can be specified when required. The message is spoken by a synthetic voice.
    At least one recipient must be specified.

    Pager Call

    Notifies the specified users by activating their pagers.
    Note:
    At least one recipient must be specified.

    FTP Message

    Sends a message file via FTP to the specified users.
    Note:
    The contents of this file is determined by the event causing the notification but a common prolog and epilog text can be specified when required.
    At least one recipient must be specified.

    Printer Message

    Prints a message on the printers of the specified users.
    Note:
    The contents of this message is determined by the event causing the notification but a common prolog and epilog text can be specified when required.
    At least one recipient must be specified.
  5. Enter a prologue text into the Prolog edit field.
    Note:
    Adding such a text is not mandatory but a form of address like for example 'Hi Mr. Smith.' makes the notification more readable or understandable.
  6. Enter an epilogue text into the Epilog edit field.
    Note:
    Adding such a text is not mandatory. Possible examples that can be used are 'Bye, bye' or 'Thanks'.
  7. Choose the report behavior by selecting either the Nothing, Everything or Failures radio button:

    Option

    Description

    Nothing

    No errors or informational messages are reported.

    Everything

    Any notification related action or problem is reported.

    Failures

    Only failures are reported.
  8. Specify the users to receive the notification by pushing the Add... button and by selecting the users in the User Addresses Dialog Box.
  9. Push the Details... button in order specify further notification details through the related dialog boxes:
    E-Mail: E-Mail Details Dialog Box
    FAX: FAX Details Dialog Box
    Phone Call: Phone Call Details Dialog Box
    Pager Call: Pager Call Details Dialog Box
    FTP Message: FTP Message Details Dialog Box
    Printer Message: Printer Message Details Dialog Box
  10. Push the Add button.
  11. Push the OK button.

Modify a Notification Profile

  1. Open the System Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Configuration panel of the System ribbon bar category.
  2. Select the Notifications page.
  3. Select the name of the notification profile to be modified in the Name combo box and perform the required changes.
  4. Push the OK button.

Delete a Notification Profile

  1. Open the System Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Configuration panel of the System ribbon bar category.
  2. Select the Notifications page.
  3. Select the name of the notification profile to be deleted in the Name combo box.
  4. Push the Remove button. Use the Remove All button in order to delete all existing notification profiles.
    Note:
    The Default notification profile cannot be deleted.
  5. Push the OK button.

Monitor the Notification Job Queue

  1. Open the System Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Configuration panel of the System ribbon bar category.
  2. Select the Notifications page.
  3. Push the Abort... button. This opens the Abort Notifications Dialog Box where inside the List list box all notification jobs are enumerated that are currently executing or waiting to be executed. Close this dialog box without aborting a notification job by pushing the Cancel button.
  4. Push the OK button.

Delete a Notification Job from the Queue

  1. Open the System Settings Dialog Box if it is not open yet:
    Push the Settings button inside the Configuration panel of the System ribbon bar category.
  2. Select the Notifications page.
  3. Push the Abort... button. This opens the Abort Notifications Dialog Box where inside the List list box all notification jobs are enumerated that are currently executing or waiting to be executed. Select only those jobs that should be aborted and then push the OK button.
  4. Push the OK button.

Note:
In some cases the abortion of notification jobs has no immediate effect. This happens in situations where a reschedule of the execution queue happened while the Abort Notifications Dialog Box was open.

Prerequisites:
The privilege 'Creating and modifying notification profiles' must be enabled in order to create, modify or delete notification profiles.
