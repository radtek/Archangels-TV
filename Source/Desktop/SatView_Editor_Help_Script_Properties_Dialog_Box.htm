Script Properties Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ provides an interface to customize the properties for an automation script which include the type of application to be automated and also the input file and output file involved. Because some scripts may require access to telemetry parameters it is also possible, for testing purposes, to associate values and other attributes with telemetry parameters.

How do I...

Configure the Properties of an Automation Script

  1. Open the Script Properties Dialog Box if it is not open yet:
    Push the Properties button inside the Details panel of the View ribbon bar category.
  2. Check either the Excel, Word, Access, Outlook or PowerPoint radio button depending on the application to be automated.
    Mark the Other button if another (not listed) application is automated.
  3. Enter the file name of the input file (if any) into the Input File edit field.
    Note:

    This file name will be assigned to the variable InputFile (see Automation Scripts for more information) and can be used within the automation code.

  4. Repeat the same for an eventual output file (if any) by typing its path name into the Output File edit field.
    Note:

    This file name will be assigned to the variable OutputFile. Furthermore, this file is considered to be the resulting file of the automation process and is displayed in the lower pane of the display window when testing the script.
  5. Push the OK button.

Related Topics:

Simulating Telemetry Conditions

When testing an automation script it sometimes might be necessary to simulate certain 'health conditions' which can be reflected by telemetry data. This can be achieved by initializing the affected telemetry parameters in the required way.

  1. Open the Script Test Parameters Dialog Box if it is not open yet:
    Push the Properties button inside the Details panel of the View ribbon bar category and push the Parameters... button in the dialog box.
  2. Select a telemetry parameter to be associated with a value in the Tag combo box.
  3. Enter the desired value into the Value edit field.
    Notes:
    • For all telemetry parameters having a textual calibration table attached the possible values are listed in the combo box part of the Value edit field.
    • Double-clicking onto a telemetry parameter with an attached calibration table in the List list box displays the calibration table through the Script Test Parameter Details Dialog Box turning-up.
    • The calibration curve used can be selected in the Table combo box.
    • A non-calibrated value can be entered by marking the Raw check box.
    • Adjust the number in the Sample spin box accordingly.
    • Mark the Past check box in order to indicate that the current value sample is a historical one.
  4. Specify the out-of-limit status, the validity and the data quality of the supplied value by checking the associated radio buttons.
  5. Push the Add button.
    Repeat the previous steps (except for the first one) as many times as telemetry parameters need to be initialized before the testing takes place.
    Note:

    Use the Set button in order to modify a telemetry parameter value. In addition, the Remove or Remove All buttons can be used to delete a selected telemetry parameter (or all) from the list of initializations.

  6. Push the OK button.

Prerequisites:
The privilege 'Creating and modifying automation scripts' must be enabled to change the properties of automation scripts.
