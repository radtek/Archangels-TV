With the Variables property page the local variables of a telecommand procedure are defined.

How do I...

Define the local Variables

A variable is defined and used within the telecommand procedure to store temporary values. It is only available within the current telecommand procedure. There may be zero or more local variables. A variable can be given a default value that is used as the initial value of the variable when the telecommand procedure starts to execute. Note that if the variable is not given a default value, it will not have a value when the procedure begins to execute. Before a telecommand procedure refers to the value of a variable, its value must be set. A variable can be given a value by a 'Set-Variables' action-object step, or by prompting an operator to provide a value though the use of a 'User-Interaction' action-object step.
Important: Arguments and variables for a particular telecommand procedure must have unique, mutual exclusive names.

Follow these points:

  1. Select the Variables property page.
  2. Fill-in the required information:

    Defines a maximum value for the variable.
    Note:
    In the case of a variable with coding 'String' a lexicographical comparison is performed in order to check against a maximal value.

    Value Interpretation

    Choose the appropriate value interpretation options:

    OptionMeaning

  3. Save the modifications by selecting the Save item on the application button's menu.

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify a telecommand procedure.
