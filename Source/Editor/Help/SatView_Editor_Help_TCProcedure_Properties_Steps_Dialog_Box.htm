TC Procedure Steps Property Page

SUMMARY

Saturday, October 10, 2015
Version 1.0

The Steps property page is available for supplying property information about the flow-control steps and action-object steps building-up the threads of a telecommand procedure.

How do I...

Configure the Flow-Control Steps & Action-Object Steps

A telecommand procedure contains one or more threads, each of them consisting of one or more steps. These steps are divided into two types: Flow-control steps on one hand and action-object steps on the other one. Flow-control steps provide the means of implementing multiple paths through a procedure, usually based on a logical test. This step type performs the role of programming constructs such as 'repeat', 'while', 'for', 'case' as well as others. Flow-control steps also have an associated invariant condition, a logical expression that must remain true throughout the execution of the step or the execution phase will fail. An action-object step carries out some kind of action as the step executes.
For example:

  • Send a telecommand
  • Generate an interaction with the operator
  • Set the value of a telecommand parameter

Each of the steps is divided into three different phases: Trigger, body and confirmation. Every step has the same configuration available for its trigger and confirmation phases but those phases are optional. The body phase is the prime function of the step and has a different configuration depending on the step. Inside the flowchart the tree step phases are symbolized by a small triangle pointing into the direction of the execution. They change their color depending on the (successful) execution of the associated phase: Green for success, red for a failure, yellow for the current execution or black for a forced result.
The following steps are supported:

Type

Step

Function

Flow-control

Branch

Provides a branch facility commonly referred to as a ‘IF-THEN-ELSE’ statement. The 'true' execution branch (sub-thread) will be executed if the branch expression evaluates to true; otherwise the 'false' sub-thread will be executed. Providing no expression always results in the execution of the 'true' sub-thread.

 

Switch

This step corresponds with a 'CASE' statement. An expression can be supplied for each branch in order to determine which one should be executed. If more than one expression results to true, the first one i.e. its associated branch is selected for execution. Providing no expression for a particular branch always results in its execution.

 

For-Loop

Implements a loop block executing a specified number of times. It corresponds with a 'FOR' statement.

 

While-Loop

Implements a loop block executing as many times as a supplied expression results to true. It corresponds with a 'WHILE' statement.

 

Do-Loop

Implements a loop block executing as many times as a supplied expression results to true. In contrast to the previous loop the expression is tested after executing its associated block. It corresponds with a 'DO' statement.

 

Block

Allows one or more steps to be grouped into a block and to be collapsed/expanded inside the flowchart.

 

Goto

Redirects the execution path to a specified label. It corresponds with a 'GOTO' statement.
Note:
This step type has no execution phases.

 

Label

Marks the entry point after a jump.
Note:
This step type has no execution phases.

Action-object

Set-Arguments

Sets or modifies the values of the procedure arguments (if any).

 

Set-Variables

Sets or modifies the values of the procedure variables (if any).

 

Set-Parameters

Sets or modifies the values of the telecommand parameters (if any).

 

Set-Synchronization Object

Signals a named synchronization object and releases any thread waiting for it.

 

Inject-Alert

Injects an alert in form of an event into the Global Eventbox.

 

Call-Procedure

Calls another telecommand procedure and waits for its completion.

 

Start-Procedure

Starts another telecommand procedure but continues its own execution.

 

Control-Procedure

Controls the execution state of the specified telecommand procedure.

 

Control-Thread

Controls the execution state of the specified thread inside the current telecommand procedure.

 

Wait-Time Interval

Waits the specified number of milliseconds.

 

Wait-Synchronization Object

Waits for a named synchronization object to get signaled or until the specified (optional) timeout (in milliseconds) expires.

 

Send-TC Function

Sends the specified telecommand function to the release queue.

 

Send-TC Sequence

Sends the specified telecommand sequence to the release queue.

 

Log-Message

Logs the specified message to a definable file.

 

User-Interaction

Requests (predefined) values for procedure variables from the operator.

Follow these points:

  1. Select the Steps property page.
  2. Fill-in the required information:

    Note:

    Description of the step.
    Note:
    This text is displayed inside the flowchart and should therefore not be too long.

    Comments

    Contains a more detailed description of the step.

    No

    Trigger Information

    Specifies a certain condition that must be fulfilled before the step starts its execution. If no trigger information is specified it always starts without any delay.

    OptionMeaning
    TypeSpecifies the type of the trigger.
    The following types are available:
    At time:Starts the execution at the time specified in the Time edit field (or later).
    Exactly at time:Starts the execution exactly at the time specified in the Time edit field. If the time has passed already when the execution reaches this step the trigger fails.
    Time offset:Waits until the number of seconds specified in the Offset spin box has expired since the creation of the thread. If already more than that amount of time has passed the execution continues immediately.
    Time delay:Waits the number of seconds specified in the Delay spin box.
    Synchronization object:Waits until the synchronization object specified in the Synchronization object combo box gets signaled. A timeout can be specified in the Time window spin box. If the synchronization object does not become signaled within the timeout period the trigger fails.
    Expression:Waits until the expression specified inside the Expression edit field becomes true or until the timeout entered with the Time window spin box expires. A timeout results in a failed trigger.
    Operator:Waits for an acknowledgment by the operator the number of seconds specified in the Time window spin box. The trigger fails if no answer is provided within that time.
    MessageAllows the attachment of a message to the trigger through the Message edit field. The text specified there is always displayed when the trigger passed successfully.
    Pre-execution expressionThe specified expression is always tested before the body phase of the step starts its execution. An empty expression results to true and forces the body of the step to be executed immediately.

    No

    Confirmation Information

    Specifies a certain condition that must be fulfilled in order to consider the step as successful. If no confirmation information is provided this phase is considered as successful.

    OptionMeaning
    TypeSpecifies the type of the confirmation.
    The following types are available:
    Command:Waits until the execution verification check for the telecommand sent by the step becomes true or until the time period specified inside the Time window expires. If a timeout occurs the confirmation is considered as failed.
    Note:
    This type can only be used for 'Send-TC Function' or 'Send-TC Sequence' action-object steps.
    Expression:Waits until the expression specified inside the Expression edit field becomes true or until the timeout entered with the Time window spin box expires. A timeout results in a failed confirmation.
    Operator:Waits for an acknowledgment by the operator the number of seconds specified in the Time window spin box. The confirmation fails if no answer is provided within that time.
    MessageAllows the attachment of a message to the confirmation through the Message edit field. The text specified there is always displayed when the confirmation passed successfully.

    No

    Failure Information

    Specifies the action that should be taken when one of the step phases fails or allows the enforcement of a certain result for each of the phases.

    OptionSteps
    Force the result of a phase
    1. Mark the Force check box.
    2. Select the phase for which a certain result should be forced in the combo box to the right of the previously mentioned check box.
    3. Select the enforced result (automatic mode = default, succeed, fail) in the right most combo box.
    Failure behaviorBy default a thread terminates when a phase of one of its steps fails. However, this behavior can be overridden for each step individually.
    1. Select the phase that requires a special failure behavior for the current step in the On failed combo box.
    2. Mark the Goto label check box in order to force the execution of the thread to continue at the selected label when the specified phase fails. Highlight the Exit thread check box to perform the default behavior.
      Note:
      The corresponding 'Label' flow-control step must already exist in the flowchart in order to be selectable.

    No
  3. Save the modifications by selecting the Save item on the application button's menu.

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify a telecommand procedure.
