Database Import Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ provides an interface to import complete databases or only parts of it. This is a useful feature helping to synchronize databases.

How do I...

Import Databases

  1. Open the Database Import Dialog Box if it is not open yet:
    Select the Import Database... item on the application button's menu.
  2. Choose the server on which the database to be imported is located by selecting it in the Server combo box.
  3. Determine the spacecraft to which the database belongs to in the Spacecraft combo box.
  4. Select one of the databases available for the desired spacecraft on the specified server by highlighting it in the Database combo box.

    Importing a complete database

    This procedure imports a complete database with a different name.

    1. Enter the name of the imported database into the Import to edit field inside the Destination group box.
    2. Specify the directory where the files of the database should be moved to by typing its path into the Location edit field.
      Note:
      Use the Browse... button to search for a directory.
    3. Push the Details... button to specify some more details about the database.
      The DBMS Settings Dialog Box will appear:

      Configuring & Optimizing the DBMS Settings

      1. Check the Data radio button in order to indicate that all following information is related to the data file of the database.
      2. Enter the name of the file into the Filename edit field.
      3. Specify the directory where the file should be located by typing it into the Location edit field. Use the Browse... button in order to find a suitable directory.
      4. Determine the initial, maximal and grow size of the file by entering the various sizes (in MB) into the associated Initial Size, Maximal Size and Grow Size edit fields.
        Note:
        Mark the Unlimited check box if a maximal size for the file is not applicable. Use the Unchangeable check box if the selected database file should be limited in its growth.
        The Read-only check box can be marked if the database should be 'read-only'.
      5. Check the Logs radio button and repeat the last steps (except for the first one) in order to specify the same information for the logs file of the database.
      6. Push the OK button.

      Browsing the Database Tables

      1. Push the Tables... button to view the relational tables to be created for the database. The Database Tables Dialog Box will appear:
      2. Browse through the tables by selecting them in the Table combo box. This displays their columns and format in the Content listbox.
      3. Push the OK button.
      Note:

      This option is only available for relational databases.
    4. Push the Users... button to specify the users that are allowed to access and modify the database.
      The User Accounts Dialog Box will appear:

      Specifying the Database Users

      Each SatView™ database holds a list of privileges for each user describing his permissions on that database. The set of privileges associated with a user is contained in a user account.

      1. Enter the name of the user account to be created into the Users edit field.
      2. Push the Add... button. The User Privileges Dialog Box will appear.
      3. Enter the password into the Use edit field of the Password group box.
      4. Confirm the password by filling it in again in the Confirm edit field.
      5. Select the privileges to be granted to the current user in the Rights combo box inside the Permissions group box.
      6. Push the Allow button.
        Repeat the previous two steps as many times as privileges should be granted.
      7. Push all necessary OK buttons.
    5. Push the Validity... button to specify how long the database should be considered to be valid.
      The Database Validity Dialog Box will appear:

      Specifying the Database Validity

      SatView™ databases can have an attached validity period i.e. trying to operate on an expired database will be alerted and real-time telemetry data processing (including archiving) will consequently be stopped. Retrieving historical telemetry data, however, or submitting telecommands is still be possible in those cases.

      1. Check either the Unlimited or Between t1 and t2 radio button depending on the requirement to assign an unlimited validity to the database or not. For the latter case specify t1 and t2 by typing the dates into the corresponding edit fields.
      2. Push the OK button.

    Importing the contents of another database

    This procedure imports the contents of a database into an existing one.

    1. Mark the Data only check box.
    2. Select the database to import the data inside the Import to combo box.
    Note:

    The users and the validity of an existing database cannot be modified when importing the contents of a database. Use the commands inside the Security panel of the Home ribbon bar category instead.

    Importing database files

    This procedure imports the data of database files.

    1. Mark the Data only check box as well as the Data Files check box.
    2. Enter the directory of the database files to be imported by typing the path into the Location edit field.
      Note:
      Use the Browse... button to search for the data files directory.
    3. Add a prefix string into the Prefix edit field and the file extension into the Extension edit field to form the complete file name of the data files.
      Note:
      The data files must have the following name: 'Location\PrefixTABLE.Extension' to import successfully.
      If 'Location\Prefix.Extension' forms an existing database data file, only that one is imported.
    4. Specify the string delimiter used by the data files inside the String Delimiter combo box. Do the same for the field and row delimiters by choosing the appropriate character inside the Field and Row combo box.
    5. Select the database to import the data inside the Import to combo box.
    Note:

    The users and the validity of an existing database cannot be modified when importing database files. Use the commands inside the Security panel of the Home ribbon bar category instead.
  5. Push the OK button.

Related Topics:

Import Database Component Items

The capability to synchronize distributed databases is an important feature and also results in an interface to import or export individual database items.

  1. Open the Database Component Import Dialog Box if it is not open yet:
    Select the Import è Database Component... item on the application button's menu.
  2. Enter the name of the file that contains the database component items to be imported into the Filename edit field.
    Use the Browse... button in order to search for the file.
  3. Select the items to be imported inside the list box and specify how to import them by either checking the Default or Custom radio button. By using the Custom radio button it is possible to import a single database item with another name and description then when it was exported.
    Note:
    Importing a database item with the same name than an existing one will overwrite it.
  4. Push the OK button.

Export Database Component Items

The capability to synchronize distributed databases is an important feature and also results in an interface to import or export individual database items.

  1. Open the Database Component Export Dialog Box if it is not open yet:
    Select the Export è Database Component... item on the application button's menu.
  2. Choose the database component items to be exported in the Directory list box.
  3. Enter the name of the file to contain the database items to be exported into the Filename edit field.
    Use the Browse... button in order to find an existing file.
  4. Specify how to export the items by either checking the Default or Custom radio button. By using the Custom radio button it is possible to export a database single item with a different name and description.
  5. Push the OK button.

Prerequisites:
The privileges 'Creating databases' or 'Creating (remote) databases' must be enabled to import databases.
