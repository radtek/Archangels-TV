Compilers Basics

SatView™ uses third-party software to generate executable modules from certain database component items. There are many ways available to control how these applications operate.

How do I...

Configure the Compile Options

While compiling database component items third-party software is used to generate executable modules. By configuring the compile options it is possible to customize that process.

  1. Open the Compile Options Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Options button inside the Settings panel.
  2. Enter the path of the executable file to be used as compiler into the Compiler edit field.
    Use the Browse... button in order to locate the file if necessary.
  3. Type the path of the linker program into the Linker edit field.
    Note:
    Mark the 32-bit check box to indicate that the specified compiler and linker are targeting a 32-bit platform (the default is 64-bit).
  4. Specify the directory where the necessary include files can be found by entering that location into the Include Directory edit field.
    Eventually use the Browse... button in order to locate the directory.
  5. Do the same for the location where all library files reside by typing it into the Library Directory edit field.
    Again use the associated Browse... button if necessary.
  6. Push the OK button.

Prerequisites:
The privilege 'Changing the database compile options' must be enabled to configure the compile options.

Specify Compile Directives

Compile directives profiles are a useful tool to suppress certain messages and to further customize the error & warning behavior during compilation.

Creating a new Compile Directives Profile

  1. Compile the database components first before creating or changing the compile directives.
    The resulting messages will appear in the Build tab of the Output tool window.
  2. Open the Compile Directives Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Directives button inside the Settings panel.
  3. Enter the name of the directives profile to be created into the Profile edit field.
  4. Select those messages in the Messages list box that need to be suppressed in future compilations when this profile is applied and then push the Disable button.
    Note:
    When messages need to be re-enabled they must be selected in the Disabled list box first before using the Enable button.
    Use the Delete button to remove a disabled message which does not need to be re-enabled any more because the cause for its occurrence has been resolved.
    Mark the Do not show any warnings check box if no warnings should be listed when a database is compiled using the selected compile directives profile.
  5. Push the Add button.
    Repeat the last three steps in order to add more than one compile directives profile.
  6. Push the OK button.

Deleting a Compile Directives Profile

  1. Open the Compile Directives Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Directives button inside the Settings panel.
  2. Select the name of the directives profile to be deleted inside the Profile combo box.
  3. Push the Remove button.
    Use the Remove All button in order to delete all compile directives profiles.
    Note:
    The Default compile directives profile cannot be deleted.
  4. Push the OK button.

Applying a Directives Profile to a Compile Pass

  1. Open the Compile Directives Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Directives button inside the Settings panel.
  2. Select the name of the directives profile to be applied to the next compilation pass inside the Profile combo box.
  3. Push the OK button.

Use Compile Batch Profiles

Parse or compile batch profiles are a convenient tool to manage different database verification and translation processes. Their execution can be scheduled on demand.

Creating a new Batch Profile

  1. Open the Compile Batches Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Batches button inside the Compilation panel.
  2. Enter the (unique) name of the batch profile to be created into the Profile edit field.
  3. Inside the Properties group box select the database components to be parsed or compiled with this batch profile by marking the appropriate check boxes.
  4. Specify whether a full compilation or only a syntax check should take place on the selected database components by checking either the Compilation or Syntax Check radio button inside the Action group box.
  5. Decide whether the batch profile should be executed immediately, at a certain time or daily by checking either the Immediately, At Time t or Daily at t radio button inside the Schedule group box.
  6. Push the Add button.
    Repeat the previous steps (except for the first one) if more than one batch profile should be created.
  7. Push the OK button.

Modifying a Batch Profile

  1. Open the Compile Batches Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Batches button inside the Compilation panel.
  2. Select the name of the batch profile to be modified inside the Profile combo box and perform the necessary changes.
    Repeat this step as many times as profiles need to be modified.
  3. Push the OK button.

Deleting a Batch Profile

  1. Open the Compile Batches Dialog Box if it is not open yet:
    Select the Build ribbon bar category and push the Batches button inside the Compilation panel.
  2. Select the name of the batch profile to be deleted inside the Profile combo box.
  3. Push the Remove button. Use the Remove All button in order to delete all compile batches.
    Note:
    The Default batch profile cannot be deleted.
  4. Push the OK button.

Prerequisites:
The privilege 'Changing the database compile batches' must be enabled to create, modify or delete compile batch profiles.

Interpreting Messages

The compilers can issue errors, warnings or other informational messages.
Depending on the pass their syntax is structured as follows:

Pass

Description

1

Syntax check related to language specific features:
The error/warning messages are reported by indicating the line number (starting at the beginning of the source code) and column position where the problem was detected.

2

Transformation of the source code into an intermediate file:
Error/warning messages are indicated the same way as in the first pass.

3

Compilation of the intermediate file:
Error/warning messages are indicated by reporting the name of the derivation procedure, mimics object procedure or telecommand procedure containing the bug. An additional line numbering allows a correct location of the problem inside the procedure.

4

Linking of the resulting binary files:
Errors/warnings are reported as linker messages.
