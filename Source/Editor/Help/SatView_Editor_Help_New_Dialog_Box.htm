New Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

SatView™ requires databases in order to interpret the telemetry data transmitted to ground, to know how to present the processed data or to generate telecommands. More than one database can be created for a certain spacecraft but only one can be dedicated the global database.
This dialog box provides an interface to create new databases or database components. The component types available depend on the database standard used.

How do I...

Create a new Database

  1. Open the New Dialog Box if it is not open yet:
    Select the New... item on the application button's menu.
  2. Select the Database item in the Type combo box in order to request a new database to be created.
  3. Type the name of the database into the Name edit field inside the Database Properties group box.
  4. Select or enter the name of the spacecraft this database is dedicated to in the Spacecraft combo box.
  5. Choose the server on which the database should be created by selecting its name in the Server combo box.
  6. Specify the scope of the database by choosing between the Global (only one per spacecraft) or Local (multiple instances allowed) radio button in the Characteristics group box.
  7. Choose one of the supported database standards by selecting the appropriate items in the Standard as well as in the Implementation combo boxes.
  8. Type the name of the binary file becoming the compiled database image into the Filename edit field inside the Files group box. Use the ... button in order to find an appropriate directory.
  9. Push the Details... button to specify some more details about the database. The DBMS Settings Dialog Box will appear:

    Configuring & Optimizing the DBMS Settings

    For the Microsoft® SQL Server®:

    1. Check the Data radio button in order to indicate that all following information is related to the data file of the database.
    2. Enter the name of the file into the Filename edit field.
    3. Specify the directory where the file should be located by typing it into the Location edit field. Use the Browse... button in order to find a suitable directory.
    4. Determine the initial, maximal and grow size of the file by entering the various sizes (in MB) into the associated Initial Size, Maximal Size and Grow Size edit fields.
      Note:
      Mark the Unlimited check box if a maximal size for the file is not applicable. Use the Unchangeable check box if the file should be limited in its growth and Bulk Copy if this kind of data import should be allowed for this database.
    5. Check the Logs radio button and repeat the last steps (except for the first one) in order to specify the same information for the logs file of the database.
    6. Push the OK button.

    Managing the Database Tables

    1. Push the Tables... button to view the relational tables to be created for the database. The Database Tables Dialog Box will appear:

      Setting-up the Tables and their Relationships

      • This feature is not yet implemented.

      Importing Tables

      • This feature is not yet implemented.

      Exporting Tables

      • This feature is not yet implemented.
    2. Push the OK button.
    Note:

    This option is only available for relational databases.
  10. Push the Users... button to specify the users that are allowed to access and modify the database. The User Accounts Dialog Box will appear:

    Creating a new User Account

    Each SatView™ database holds a list of privileges for each user describing his permissions on that database. The set of privileges associated with a user is contained in a user account.

    1. Enter the name of the user account to be created into the Users edit field.
    2. Push the Add... button. The User Privileges Dialog Box will appear.
    3. Enter the password into the Use edit field of the Password group box.
    4. Confirm the password by filling it in again in the Confirm edit field.
    5. Select the privileges to be granted to the current user in the Rights combo box inside the Permissions group box.
    6. Push the Allow button.
      Repeat the previous two steps as many times as privileges should be granted.
    7. Push all necessary OK buttons.

    Modifying a User Account

    1. Select the user those account password or permissions on the database is going to be changed in the Users combo box.
    2. Push the Change... button. The User Privileges Dialog Box will appear and then change either the user's password and/or privileges.
    3. Push all necessary OK buttons.

    Deleting a User Account

    1. Select the user those account has to be deleted in the Users combo box.
    2. Push the Remove button.
    3. Push the OK button.
  11. Push the Validity... button to specify how long the database should be considered to be valid. The Database Validity Dialog Box will appear:

    Specifying the Database Validity

    SatView™ databases can have an attached validity period i.e. trying to operate on an expired database will be alerted and real-time telemetry data processing (including archiving) will consequently be stopped. Retrieving historical telemetry data, however, or submitting telecommands is still be possible in those cases.

    1. Check either the Unlimited or Between t1 and t2 radio button depending on the requirement to assign an unlimited validity to the database or not. For the latter case specify t1 and t2 by typing the dates into the corresponding edit fields.
    2. Push the OK button.
  12. Push the OK button.

Prerequisites:
The privileges 'Creating databases' or 'Creating (remote) databases' must be enabled.

Create a new Database Component Item

  1. Open the New Dialog Box if it is not open yet:
    Select the New... item on the application button's menu.
  2. Select the component to be created in the Type combo box.
    Most database standards support at least the following components:

    Component

    Type

    Description

    TM Packet

    Core DBMS

    Creates a new telemetry packet.

    TC Function

    Core DBMS

    Creates a new telecommand function.

    TC Sequence

    Core DBMS

    Creates a new telecommand sequence.

    TC Procedure

    DLL

    Creates a new telecommand procedure.

    TM Parameter

    Core DBMS

    Creates a new telemetry parameter.

    TC Parameter

    Core DBMS

    Creates a new telecommand parameter.

    On-Board Processor

    Core DBMS

    Creates a new on-board processor.

    Alphanumeric Display

    Core DBMS

    Creates a new alphanumeric display.

    Graphic Display

    Core DBMS

    Creates a new graphic display.

    Mimics Display

    DLL

    Creates a new mimics display.

    Parameter Observation Display

    Core DBMS

    Creates a new parameter observation display.

    Automation Script

    DLL

    Creates a new automation script.
  3. Push the OK button.

Prerequisites:
The corresponding privilege must be enabled to create, modify or delete a database component item.
See Privileges Summary for all existing privileges.
