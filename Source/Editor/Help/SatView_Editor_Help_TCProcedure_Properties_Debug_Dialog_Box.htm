TC Procedure Debug Property Page

SUMMARY

Saturday, October 10, 2015
Version 1.0

The Debug property page is available for configuring the debugging options for a telecommand procedure.

How do I...

Configure the debugging Options

Debugging is an important step in the process of developing a telecommand procedure.
The following features are supported to help testing:

BreakpointsSuspend the execution when a breakpoint is reached.
Breakpoints can be set at each (named) step or source code and can have a conditional nature.
Test ProfilesAllow the definition of telemetry parameter groups attached with a predefined value. This is useful to simulate various spacecraft configurations and can also be used in the context of the telecommand pre-execution verification and execution verification checks. Test profiles can be injected at each (named) step.
IdentifiersWhenever the execution is suspended either by a breakpoint or manually all arguments, variables, telemetry parameters and telecommand parameters used inside the telecommand procedure are listed and their values can be modified when necessary.

Follow these points:

  1. Select the Debug property page.
  2. Configure whatever is required:

    Setting a Breakpoint

    1. Select the thread in the Thread combo box that should get a breakpoint.
    2. Inside the Properties group box specify the location for the breakpoint by choosing it in the Before step combo box.
      Note:
      Only steps that have a name can be attached with a breakpoint. Read how to configure the various flow-control steps and action-object steps in order to learn how to name a step.
    3. Specify under what circumstances a breakpoint should suspend the execution:
      AlwaysBreaks always
      At failed triggerBreaks only when the trigger phase failed
      At failed bodyBreaks only when the body phase failed
      At failed confirmationBreaks only when the confirmation phase failed
    4. Mark the Disabled check box if a breakpoint is temporarily not used but should not be deleted.
    5. Push the Add button.

    Alternative way to set a Breakpoint

    1. Select the pane (on the left part of the telecommand procedure display) referring to the thread in those execution flow a breakpoint should be set.
    2. Activate the step to which a breakpoint should be attached to by clicking onto it. It will get highlighted when it is activated.
    3. Click with the mouse button onto the left gray border inside the flowchart pane at the vertical position of the activated step.
    Note:

    Breakpoints can also be set on lines of source code by clicking at the desired location inside the debug margin of the source code pane.

    Enabling or disabling a Breakpoint

    1. Select the thread in the Thread combo box that should get a breakpoint changed.
    2. Select the breakpoint that should be enabled or disabled in the Breakpoints list view.
    3. Mark the Disabled check box to disable or uncheck it to re-enable the breakpoint.
      Note:
      Other properties of the breakpoint can also be changed this way.

    Alternative way to change a Breakpoint

    1. Select the pane (on the left part of the telecommand procedure display) referring to the thread in those execution flow a breakpoint should be changed.
    2. Right-click with the mouse button onto the breakpoint symbol inside the left margin of the flowchart pane.
    3. Check the Disable menu item in the appearing floating menu to disable or uncheck it to re-enable the breakpoint.
      Note:
      Other properties of the breakpoint can also be changed this way.

    Removing a Breakpoint

    1. Select the thread in the Thread combo box that should get a breakpoint deleted.
    2. Select the breakpoint that should be deleted in the Breakpoints list view.
    3. Push the Remove button to delete it. Use the Remove All button to delete all breakpoints.

    Alternative way to delete a Breakpoint

    1. Select the pane (on the left part of the telecommand procedure display) referring to the thread in those execution flow a breakpoint should be deleted.
    2. Click onto the breakpoint to be deleted inside the left margin of the flowchart pane.

    Creating a Test Profile

    1. Select the thread in the Thread combo box that should make use of a test profile.
    2. Enter the name of the test profile to be created into the Name edit field located inside the Properties group box associated with the test profiles.
    3. Add all the telemetry parameters to be included in the test profile by pushing the Add button and by filling-in all relevant information into the appearing TC Procedure Test Profile Parameters dialog box:

      Specifying Parameters for a Test Profile

      1. Select a parameter inside the Parameter combo box.
        Note:
        The default value interpretation for the selected parameter (as defined in the database) is automatically shown inside the Value Interpretation group box.
      2. Enter a value corresponding with the value interpretation into the Value edit field.
        Note:
        Some aspects of the value interpretation are modifiable. The entered parameter value, however, must always correspond with the specified interpretation.
      3. Push the Add button in order to insert the current parameter into the list and repeat the previous steps as many times as required.
        Use the Remove or Remove All button in order to delete one or all parameters from the list.
        Note:
        If a sample with an occurrence number higher than 0 or multiple samples are supplied for a parameter, they are separated by a '/' symbol.
      4. Push the OK button.
      Use the Remove button to delete a specific parameter or the Remove All button to delete all of them.
    4. Specify at what time the test profile should be injected by selecting a step in the Before step combo box that, when the execution point reaches it, triggers the test profile.
      The following complementary options can be specified:

      Option

      Description

      Immediately

      Injects the test profile immediately after the execution arrives at the specified step.

      n seconds after start

      Applies the test profile n seconds after the thread has started.

      n seconds after the start of the step

      Applies the test profile n seconds after the execution arrived at the specified step.
      Note:

      Test profiles can be attached to named steps only (except for those executing n seconds after start). Read how to configure the various flow-control steps and action-object steps in order to learn how to name a step.

    5. Push the Add button below the Test Profiles list view to add the test profile.

    Enabling or disabling a Test Profile

    1. Select the thread in the Thread combo box that contains the test profile to be enabled or disabled.
    2. Select the test profile to be modified in the Test Profiles list view and mark the Disabled check box to disable the test profile or uncheck it to re-enable the test profile.

    Deleting a Test Profile

    1. Select the thread in the Thread combo box that contains the test profile to be deleted.
    2. Select the test profile to be deleted in the Test Profiles list view and push the Remove button. Use the Remove All button to delete all of them.

    Modifying Identifier Values

    1. Start the debugging and suspend the execution at the desired location either manually or by a breakpoint.
    2. Select the identifier to be changed in the Watch list view.
    3. Enter a different value in the Value edit field. Push the Set button to update the identifier.
      Note:
      For telemetry parameters or telecommand parameters that require a calibrated value to be entered, the current calibration table is indicated in the Table combo box. Double-clicking on the identifier inside the Watch list view makes all calibration points visible.
      Both types of parameters support multiple samples via the Sample spin box and allow the specification of the out-of-limit status for the value supplied. A validity or quality indication for that value can be specified for telemetry parameters only.
  3. Save the modifications by selecting the Save item on the application button's menu.

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify a telecommand procedure.
