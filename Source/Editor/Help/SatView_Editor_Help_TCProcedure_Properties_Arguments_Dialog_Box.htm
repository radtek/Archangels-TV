How do I...

Specify the formal Arguments

When a telecommand procedure is executed, it is possible to define a number of values which are passed in to the procedure at run time. The values passed to the telecommand procedure when instantiating it for execution are referred to as arguments. Within the definition of the telecommand procedure, the number and type of arguments which the procedure takes at run time must be declared. A procedure may have zero or more arguments. In order for the telecommand procedure to execute, each argument must be given a value at the start of execution.
An argument can receive a value in a number of ways when execution of the telecommand procedure commences:

  • If the telecommand procedure is called from within another one the value of the arguments is specified by the calling procedure.
  • A telecommand procedure started manually must have argument values provided by the operator configuring the procedure instance.
  • In the case of a scheduled telecommand procedure execution the argument values can consist of predefined values but also of expressions that are evaluated when the execution begins.

Alternatively, each argument can be given a default value in the editor that will be used if a value is not given at run time.
Within a telecommand procedure the value of an argument can be referred to or changed. An expression may refer to the value of an argument or a 'Set-Argument' action-object step can give an argument a new value. Arguments are typically used to make a telecommand procedure more general. For example a spacecraft maneuvering procedure may use its arguments to represent thruster burn times.
Important: For each telecommand procedure its arguments and variables must have unique, mutual exclusive names.

Follow these points:

  1. Select the Arguments property page.
  2. Fill-in the required information:

    Field

    Description

    Mandatory

    Name

    Name of the formal argument.

    Yes

    Description

    Describes the argument in more detail.

    No

    Type

    Type of the formal argument:

    RawSelect this type if it is a raw (not calibrated) telecommand parameter or if it is not a telecommand parameter (e.g. just a string or integer value).
    NumericalSpecify this type if the formal argument is a telecommand parameter with an associated numerical calibration curve. Any default value and the actual argument value will be interpreted as calibrated values.
    StatusUse this type if the formal argument is a telecommand parameter with a textual calibration curve. Any default value and the actual argument value will be interpreted as calibrated values.

    Yes

    Default value

    Specifies a default value for the argument.

    No

    Minimum value

    Defines a minimum value for the argument.
    Note:
    In the case of an argument with type 'Raw' and coding 'String' a lexicographical comparison is performed in order to check against minimal value.

    No

    Maximum value

    Defines a maximum value for the argument.
    Note:
    In the case of an argument with type 'Raw' and coding 'String' a lexicographical comparison is performed in order to check against maximal value.

    No

    Value Interpretation

    Choose the appropriate value interpretation options:

    OptionMeaning
    RawThe default value, minimum value and maximum value are to be interpreted as raw (not calibrated) values. This option requires the type to be 'Raw' too.
    CalibratedThe supplied values (if any) are calibrated. This option requires the type of the argument to be either 'Numerical' or 'Status'.
    CodingSpecifies the coding of the argument value (including default value, minimum value and maximum value).
    RadixSpecifies the radix of the argument value (including default value, minimum value and maximum value).
    TableFor any calibrated value this combo box specifies the calibration table used to de-calibrate the supplied values.

    Yes
    Note:

    Push the Add button in order to register an argument or use the Remove button to delete a selected argument from the List list view summarizing all arguments. Remove All clears the complete list.

  3. Save the modifications by selecting the Save item on the application button's menu.

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify a telecommand procedure.
