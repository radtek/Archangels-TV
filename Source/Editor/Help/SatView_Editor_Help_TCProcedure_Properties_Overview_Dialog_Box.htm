TC Procedure Overview Property Page

SUMMARY

Saturday, October 10, 2015
Version 1.0

The Overview property page is available for supplying basic information on a telecommand procedure.

How do I...

Specify general Information

  1. Select the Overview property page.
  2. Fill-in the required information:

    Field

    Description

    Mandatory

    Name

    Name of the telecommand procedure.
    Note:
    It must be unique in order to clearly identify the procedure.

    Yes

    Description

    Description of the telecommand procedure.

    Yes

    Comments

    Contains a more detailed description of the telecommand procedure.

    No

    Author

    Identifies the author of the telecommand procedure.

    No

    Creation date and time

    Indicates the date and time when the telecommand procedure was created.

    No

    Last modified

    Indicates the date and time when the telecommand procedure was modified the last time.

    No

    Destination sub-system

    Identifies the sub-system on-board of the spacecraft to which any telecommands are related to.

    No

    Default execution mode

    Defines how the telecommand procedure should be executed:

    AutomaticThe procedure executes in an automatic way. This is the default if nothing is specified.
    Single StepThe procedure executes pausing at each step for operator authorization.
    External ActionsThe procedure executes pausing at each external action-object step for operator authorization. External actions are sending telecommands, setting arguments, variables, parameter values and calling/starting other telecommand procedures.
    Note:
    This execution mode is useful to acknowledge or eventually modify the values associated with the action-object step.
    CommandsThe procedure executes pausing at each action-object step related to sending a telecommand for operator authorization.
    Note:
    This execution mode is useful to acknowledge or eventually modify the current parameter values before sending a telecommand.

    No

    Verification status

    Indicates if the telecommand procedure has been tested and whether it is operational or not.

    No

    No graphical interface

    Provides the possibility to develop code for a telecommand procedure without the graphical interface.

    No
  3. Save the modifications by selecting the Save item on the application button's menu.

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify a telecommand procedure.
