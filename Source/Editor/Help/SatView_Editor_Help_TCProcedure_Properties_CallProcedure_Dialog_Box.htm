Call-Procedure Step Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

This action-object step calls another telecommand procedure and waits for its completion.

How do I...

Configure the Call-Procedure Step

  1. Open the Call-Procedure Step Dialog Box if it is not open yet:
    Double-click onto the step to be configured or activate it and select the Properties button on the toolbar.
  2. Select the telecommand procedure to be called in the Procedure combo box.
  3. Enter an optional description about what actions are performed with this telecommand procedure call into the Description edit field.
  4. Specify the invariant condition that must be true before the telecommand procedure is called and enter it into the Invariant Condition edit field.
    Note:
    If no condition is specified the default results to true.
  5. Set a value for all arguments by selecting each of them (one after the other) inside the list box of the Arguments group box and by entering its value in the Value edit field. Confirm the value for each argument by pushing the Set button.
    Note:
    Be careful to enter a value corresponding with the coding and radix of the argument. In particular, any radix other than decimal will require a constant (with the specified radix) to be specified and not an expression.
  6. Mark the Requires Authorization check box if an authorization is required (and also if some argument values eventually need to be checked or modified manually) before proceeding with the execution.
  7. Push the OK button.

Any condition or expression must be expressed in TPL and evaluate to either true or false.
See these examples:

  • GetValue(R328) == 800
  • GetRawValue(K600) == 1 && GetValue(G220) == TEXT("ACQUISITION")

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify telecommand procedures.
