TC Procedure Stack Property Page

SUMMARY

Saturday, October 10, 2015
Version 1.0

The Stack property page shows all output generated by a telecommand procedure. This includes all released telecommands but also those waiting for release.

How do I...

Monitor & Control the Telecommand Stack

The final purpose of telecommand procedures is, of course, the generation of telecommands. All telecommands that are scheduled and released by the procedure are listed in a stack together with some status information. This feature also supports the pre-execution verification checks and the execution verification checks. By logging the output of the release queue to a file, extended control about the correct functioning of the telecommand procedure can be obtained.

Follow these points:

  1. Select the TC Stack property page.
  2. Choose the appropriate action:

    Browsing the Telecommand Stack

    • All scheduled and released telecommands are listed in the Telecommands list view with the following formatting:

      Column

      Description

      No.

      Displays the entry number of a telecommand on the stack.

      Name

      Identifies the telecommand.
      If it is part of a telecommand sequence the name of the sequence is appended in brackets.

      Description

      Describes the telecommand in more detail.

      Arguments

      Shows all non-fixed arguments of the telecommand together with their current value.
      Note:
      Not all arguments require a value. Some argument values are created automatically.

      Flags

      Indicates if a telecommand is grouped with the previous entry ('G') or if it is part of a block ('Bn' where n is the entry number in the block).

      TC Packet

      Shows the telecommand packet with which the current telecommand is encoded.

      Release Time

      Shows the time when the telecommand was released.

      Rel. Offset

      Indicates the release time offset with respect to the start of the telecommand sequence. It is used to calculate the absolute release time.

      Execution Time

      Shows the absolute on-board execution time of the telecommand.

      Exec. Offset

      Indicates the on-board execution time offset with respect to the start of the telecommand sequence. It is used to calculate the absolute execution time.

      Authorization

      Indicates the type of authorization required.

      Status

      Shows the current release status and verification status of the telecommand.

      Result

      Shows the result of the various checks or if the release of the telecommand was aborted.

    Suspending or Resuming the Telecommand Release Queue

    1. Push the Suspend button at the desired time i.e. when there are telecommands in the release queue.
      Note:
      Suspending the telecommand stack will prevent any telecommands from being released. Execution verification checks however will still continue to be done.
    2. Push the Resume button in order to continue the scheduling of the telecommand stack.
    Note:

    Suspending the telecommand stack for a too long time may result in expired telecommands.

    Manipulating the Contents of the Telecommand Release Queue

    • Select the scheduled telecommand to be removed from the release queue and push the Remove button. Use the Remove All button to delete all scheduled telecommands.
    Note:

    Released telecommands obviously cannot be removed from the stack because they not part of the release queue any more.

    Enabling or Disabling the Release Wait Mode

    • Mark the Release Wait Mode check box in order to enable telecommands to wait for their release in case the telecommand pre-execution condition is not satisfied. The number of seconds to wait can be specified in the associated spin box.
    Note:

    This option does not apply to the on-board execution time of telecommands.

    Enabling or Disabling the Pre-execution Verification Checks

    • Mark the Disable pre-execution verification checks check box in order to disable any pre-execution verification checks.

    Enabling or Disabling the Execution Verification Checks

    • Mark the Disable execution verification checks check box in order to disable any execution verification checks.

    Logging all Output to a File

    • Mark the Log to file check box and enter a file name into the associated edit field. All released telecommands will be written to the specified file the next time the telecommand procedure is tested.

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify a telecommand procedure.
