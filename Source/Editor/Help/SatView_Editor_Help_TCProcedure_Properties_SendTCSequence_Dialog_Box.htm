Send-TC Sequence Step Dialog Box

SUMMARY

Saturday, October 10, 2015
Version 1.0

This action-object step sends the specified telecommand sequence to the release queue.

How do I...

Configure the Send-TC Sequence Step

  1. Open the Send-TC Sequence Step Dialog Box if it is not open yet:
    Double-click onto the step to be configured or activate it and select the Properties button on the toolbar.
  2. Select the telecommand sequence to be released in the TC Sequence combo box.
  3. For each telecommand of the sequence, select its entry number in the Entry combo box and perform the subsequent steps:
    1. Modify the description of the current telecommand in the Description edit field (if required).
    2. Mark the Release Time check box and enter the requested absolute release time for the telecommand or use the Offset option in order to specify an offset (in seconds) with respect to the first telecommand of the sequence. However, in case that one is of immediate release type, the offset is added to the time when this step is released.
      Note:
      If the entry is associated to a telecommand procedure (instead of a normal telecommand) the release time is used as the time when the procedure is scheduled to be executed.
    3. Mark the Execution Time check box and enter the on-board execution time in the edit field if a time-tagged telecommand should be generated or use the offset option (right to the absolute execution time) in order to specify an offset (in seconds) for the execution time.
      Note:
      Whenever an on-board execution time offset is specified, the absolute execution time will be calculated based on either the on-board execution time of the first entry or, if not available, the dispatch time of the first sequence entry (+offset).
    4. Mark the Group check box to indicate that the selected entry should be grouped with the previous one when it is released i.e. without any other intermittent telecommands.
    5. Mark the Ignore pre-execution verification check box if the predefined pre-execution verification checks associated with the telecommand function should be ignored.
      An alternative pre-execution verification expression can be entered in the associated edit field if this option is selected.
    6. Mark the Ignore execution verification check box if the predefined execution verification checks associated with the telecommand function should be ignored.
      An alternative execution verification expression can be entered in the associated edit field together with a time window (timeout for the expression to become true) if this option is selected.
      Note:
      The alternative execution verification expression is always related to the completion verification stage. It should not be used to indicate the success or failure of other verification stages.
    7. Mark the Critical check box if the selected telecommand requires a confirmation before being released.
      The Requires Authorization option offers a more flexible way of authorization as it allows the modification of the non-fixed telecommand parameter values before release.
      Note:
      The Requires Authorization option is not available for telecommand procedures embedded inside a sequence.
    8. Whenever there are execution verification checks associated with the selected telecommand function the operator can be obliged to acknowledge these verification stages after completion. This is achieved by marking the Acceptance, Start, Progress and/or Completion check boxes.
    9. Set a value for all parameters of the telecommand by selecting each of them (one after the other) inside the list box of the Parameters group box and by entering its value in the Value edit field. Confirm the value for each parameter by pushing the Set button. Use the Load Values... button to initialize the parameters of a telecommand by a predefined value set. This is achieved through the TC Procedure Parameter Sets Dialog Box:

      Choosing a Telecommand Parameter Set

      • Select one of the predefined value sets to be used for the initialization of the telecommand parameters in the Name combo box.
      Note:

      The contents of the selected value set is displayed in the Contents list box. If a telecommand contains more parameters than listed by the selected set only those will be initialized that are also member of the parameter value set.
      For calibrated numerical parameters it is possible to disable or enable the extrapolation with respect to the calibration curve by marking the No extrapolation check box. This option is used when the calibrated value is converted back to a raw value. Marking the Show All check box will list all parameters with a fixed (non-modifiable) value.
      Telecommand sequences may have formal parameters. They can be made visible by marking the Show formal parameters check box. Initialization of these parameters takes place the same way as for ordinary parameters. It is advisable to associate the formal parameters with a value first before doing it for the other (normal) ones.
      Note:
      Be careful to enter a value corresponding with the coding and radix of the parameter. In particular, any radix other than decimal will require a constant (with the specified radix) to be supplied and not an expression. The calibration curve associated with some parameters can be displayed by double-clicking on those entries in the list view.
  4. Push the OK button.

Any condition or expression must be expressed in TPL and evaluate to either true or false.
See these examples:

  • GetValue(R328) == 800
  • GetRawValue(K600) == 1 && GetValue(G220) == TEXT("ACQUISITION")

Prerequisites:
The privilege 'Creating and modifying telecommand procedures' must be enabled to modify telecommand procedures.
