  • 1. Introduction

    SatView™ supports the Packet Front-End Link Protocol (PFLP) as defined by ESA in order to communicate with telemetry data provider devices. This document describes the various implementations of PFLP required to operate with the devices supported. SatView™ can be configured to act as a PFLP client or server or both (daisy-chaining).

    2.1.1. Messages

    The TT&C 3801 accepts several simultaneous connections for its telemetry services.

    In order to establish a service and to provide the telemetry services the following messages are exchanged:

    An Acknowledgment Message is sent by the TT&C 3801 after the reception of any message in which such an acknowledgment is requested. Status Messages on the other hand are transmitted periodically after a certain time-out and when no telemetry data is available.

    2.1.1.4. Service Request Message

    A Service Request Message is sent by a client whenever a telemetry service is required and needs to be initiated. With this message the endianity of the client can be checked.     2.2. ENERTEC TT&C 3801 Emulation

    Each SatView™ workstation has a built-in capability to emulate a TT&C 3801 unit and hence can act as a telemetry data server or client or both. This feature therefore enables the reception of telemetry data directly from another SatView™ workstation and allows daisy-chaining.

    The following restrictions apply in emulation mode:

    • The Transfer Frame service level is not supported.
    • SatView VX.x is returned as server description in the Service Request Response Message.
